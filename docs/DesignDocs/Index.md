# DesignDocs 索引

## 作用

本索引只登记会被频繁回看的核心设计文档，不做全目录罗列。

## 必读入口

### `ProjectNamingAndIndexing.md`

- 作用：项目命名与索引总规范
- 谁会改：规则维护者、规范治理变更
- 何时优先查看：新增文件、改共享 ID、补索引、做命名收敛前

### `CodebaseBigPicture.md`

- 作用：代码大局观，说明前后端运行主链路、模块职责边界、过重类和后续减重方向
- 谁会改：架构治理、跨模块重构执行者
- 何时优先查看：判断代码应该落在哪一层、复审耦合问题、规划结构整改前

### `UnityNamingTags.md`

- 作用：Unity GameObject 命名标签参考表
- 谁会改：前端 / 场景规范维护者
- 何时优先查看：新建场景对象、Prefab 子节点、UI 层级时

### `UnityMCPUsageRules.md`

- 作用：Unity MCP 使用规范，汇总资产层边界、默认做法、例外条件与高频默认约定
- 谁会改：Unity 协作规范维护者、场景 / UI / Prefab 治理执行者
- 何时优先查看：处理场景、Prefab、UI、SerializeField、TMP 字体和 MVC 边界问题前

### `ErrorPreventionIndex.md`

- 作用：低级错误预防索引，总结可复用错误与明细入口
- 谁会改：规则维护者、出现可复用错误后的执行者
- 何时优先查看：修类似 bug、做高风险修改、回顾历史踩坑点前

## 说明

1. 当前实现状态优先看对应 `docs/Workstreams/`，不要从执行记录或 handoff 推断。
2. 需要追溯实现原因时，从 `docs/AIChanges/ChangeIndex.md` 进入相关功能目录。
3. 架构口径变更优先看 `docs/DecisionRecords/`。
4. 若新增会被高频引用的核心文档，应回写到本索引。
