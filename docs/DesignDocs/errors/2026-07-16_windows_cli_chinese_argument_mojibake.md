# Windows CLI 中文参数进入 Python 后乱码

## 错误现象

使用 PowerShell 调用 Python 路由测试脚本并传入 `--game-time "第4天 09:38"` 后，结果 JSON 中的时间变成乱码。该参数已经进入真实 LLM prompt，因此首轮结果不能作为有效证据。

## 根本原因

Windows shell、进程参数编码和 Python 运行时编码没有形成稳定一致的 Unicode 传递链。文件内容使用 UTF-8 并不能保证命令行参数也按同一编码进入子进程。

## 正确做法

1. 优先传递语义等价的 ASCII 参数，例如 `Day 4 09:38`。
2. 必须传中文时，改用 UTF-8 配置文件或 JSON 输入文件，不依赖命令行参数编码。
3. 真实 LLM 调用前先把最终上下文回显到结果或 dry-run 产物，确认没有乱码。
