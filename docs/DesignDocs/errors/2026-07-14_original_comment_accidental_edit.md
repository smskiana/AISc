# 2026-07-14: 兼容标记误改原有注释

## 现象

为 `SaveManager` 标记旧协议兼容定位时，直接改写了模块说明和类 docstring。代码行为没有变化，但违反了项目“绝不修改或删除原有注释”的硬约束。

## 根本原因

把“更新当前工程口径”误等同于“改写代码中的既有说明”，没有先区分原有注释与本轮可新增的补充注释。

## 正确做法

1. 原有注释保持逐字不动。
2. 新定位通过独立新增注释、README、Workstream 或 ADR 表达。
3. 补丁应用后立即检查注释 diff，确认没有删除或替换旧注释行。

## 影响范围

所有带历史注释、docstring、XML summary 的代码文件，尤其是给旧类增加 deprecated / compatibility 定位时。

## 修改前自查问题

1. 这一行是本轮新增内容，还是已有说明？
2. 能否在原文前后新增补充，而不是改写原文？
3. 工程口径是否更适合写入 README、Workstream 或 ADR？
