# 持久化认知缺少查询投影导致 IDE 不可达

## 错误现象

写事务成功且快照文件包含符号或关系，但刷新 IDE 后功能树仍为空，用户无法进入已持久化数据。存储测试全部通过，却不满足产品的可达性验收。

## 根本原因

写模型允许探索结果先保存符号而不创建正式域或归属，这是正确的不变量；读模型却只投影已有正式 membership 的符号。写模型与读模型对“未归属”状态的覆盖不对称，导致合法持久状态没有 UI 入口。

## 正确做法

1. 为每个合法持久状态定义可达的查询投影，不能把“数据库中存在”当成“产品中可用”。
2. 当领域不变量禁止自动创建正式实体时，使用明确标记的派生/虚拟视图，而不是污染持久模型。
3. 虚拟节点必须稳定、可审计且非持久化；正式归属建立后应自动消失，不能复制底层实体。
4. 测试同时断言持久层未被改写和查询层可达，避免通过放宽写模型解决读模型缺口。

## 本次修复

Project Cognition 增加非持久化 `待归类` 域和 reference memberships，并让 MCP 工具、只读资源与关系查询共享同一投影。真实快照 smoke 锁定 `0` 个正式域仍可展示 `61` 个未归属符号。
