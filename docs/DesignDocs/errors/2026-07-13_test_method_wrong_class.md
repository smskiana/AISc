# 测试方法因补丁插入点落入错误类

## 错误现象

在测试文件中插入新的测试类后，原有两个测试方法被留在新类定义之后，Python 将它们归入了新测试类。运行时因新类没有原测试夹具字段而出现 `AttributeError`。

## 根本原因

补丁以某个测试方法末尾作为插入锚点，却没有同时确认原测试类后续是否还有方法。Python 依赖缩进和类声明确定归属，文本补丁成功不代表类边界正确。

## 正确做法

1. 在测试类之间插入新类前，先读取原类从声明到下一个类声明的完整范围。
2. 应优先在明确的下一个 `class` 声明前插入，而不是在任意方法末尾插入。
3. 补丁后立即运行测试发现，并检查测试全名中的类名是否符合预期。
4. 使用 `__new__` 构造测试对象时，显式补齐被测方法会访问的全部实例字段。

## 影响范围

- Python 测试文件新增或移动测试类。
- 使用 `apply_patch` 在长测试文件中按局部上下文插入代码。
