## 30 天后：衰减和融合的效果

### 关键机制：事件间边的建立

**新鲜时 — 时间节点充当连接器：**

```
  三条送面包事件各自有独立的时间锚:

  Day30送面包 ──happened_at 0.9──→ Day30_10:00
  Day35送面包 ──happened_at 0.85──→ Day35_09:30
  Day42送面包 ──happened_at 0.9──→ Day42_10:15

  事件之间: 没有直接边。检索需要经时间节点（两跳）
```

**时间衰减后 — 夜间演化时自动建立事件↔事件边：**

```
  三个时间节点 energy 降到 ~0.1（即将消失）
  → 演化引擎检测到，在删除时间节点前建立事件间边：

            Day30送面包
                │
    sequenced 0.5│similar_to 0.7
                │
            Day35送面包
                │
    sequenced 0.5│similar_to 0.7
                │
            Day42送面包

  三个时间节点随后消失。
  → 时间锚没了，但事件之间的顺序和相似关系保留了下来
  → sequenced 保留叙事链（"先...然后..."）
  → similar_to 保留同质聚类（"她每次都..."）
```

### 进一步融合 — 三周后

```
  similar_to 边反复强化（每次夜间聚类 +0.05）→ 分别达到 0.85, 0.8, 0.75
  → 三个事件形成紧密簇
  → 触发反思生成:

          ┌──────────────────────────────┐
          │  千早送面包 ×N               │ ← type=reflection
          │  "她最近每次来都有点反常。      │   energy=0.8
          │   她好像有什么话想跟我说。"     │   decay_rate=0.005
          └──────────┬───────────────────┘
                     │
        contains 0.8 │ contains 0.75   contains 0.7
                     │
    ┌────────────────┼────────────────┐
    ▼                ▼                ▼
  Day30送面包    Day35送面包     Day42送面包
  (原始事件)     (原始事件)       (原始事件)
  ↔similar_to↔  ↔similar_to↔    ↔similar_to↔
  ↔sequenced↔   ↔sequenced↔     ↔sequenced↔

  原始事件不删除，反思节点是它们的公共出口。
  日常检索先碰到反思节点（cost 低、概括性强）。
  需要细节时沿 contains 边深入原始事件。
```

### 同时：其他已消失/降级的节点

```
  ❌ 话语 "她笑得有点勉强" → energy < 0.1 → 消失
     (具体的措辞忘了，但"她不对头"的感觉保留在情绪节点)
  ⚠️ 地点 "花店.counter" → exact→vague（只记得"在花店"）
  ⚠️ 情绪 "担心" → exact→vague（"一种不太好的感觉"）
```

## 一次激活扩散：路由追踪

```
场景: 下午 14:00，下雨。玩家走进花店。
      鹿岛樱精力=55，budget≈4.2。

起点节点（场景触发）: "花店" + "下雨" + "玩家"

Step 1: 查邻边（SQLite）

  从"玩家"出发:
    → relationship "我"(0.55)  cost=1.8
    → involved 搬花盆(0.6)     cost=1.7
    → involved 修招牌(0.5)     cost=2.0
    → associated 那天躲雨(0.25) cost=4.0

  从"下雨"出发:
    → located_at 那天躲雨(0.7)  cost=1.4
    → associated 父母忌日(0.2)  cost=5.0 ← 禁区

  从"花店"出发:
    → located_at 搬花盆(0.8)    cost=1.25
    → located_at 千早送面包(0.7) cost=1.4
    → located_at 和叶拍照(0.3)  cost=3.3

Step 2: LLM 路由决策

  输入:
    budget≈4.2, 情绪=平静, 羁绊度=0.55
    策略概率: 深度38% | 最佳28% | 广度18% | 轻松16%

  路由选择:
    选了: 下雨→那天躲雨(cost 1.4) + 玩家→搬花盆(cost 1.7)
          两条路径在"搬花盆"和"那天躲雨"之间交汇
          → 它们通过 associated_with 边连在一起（玩家帮搬花盆那天也下了雨）

    预算: 4.2 - 1.4 - 1.7 = 1.1 剩余
    路径 3: 花店→千早送面包(cost 1.4) → 不够了 → 跳过

    决定: 不再深入。停在这里。

Step 3: 按选择查库

  返回节点:
    - "搬花盆"节点（完整内容）
    - "那天躲雨"节点（完整内容）
    - "感激"情绪节点（从搬花盆出发）
    - "温暖"情绪节点（从躲雨出发，但 precision=vague）

Step 4: 记忆重建 → 鹿岛樱的对话

  "下雨了呢。"
  （停顿）
  "上次下雨的时候...你也是站在门口。花盆很重，你一次就搬起来了。"
  （她没说"感激"——但玩家能感觉到）
```

## 权重不对称的实际影响

```
鹿岛樱想起九条:
  樱 → 九条: 0.15, cost=6.7
  樱的 budget 日常约 4-6
  → 几乎从不会主动想到九条
  → 除非极特殊状态（深夜 + 精力满 + 低落 → budget×1.3×1.2=约7.8）
  → 那时才能走通这条边

九条想起鹿岛樱:
  九条 → 樱: 0.2, cost=5.0
  九条的 budget 日常约 5
  → 日常也走不通
  → 但九条低落时 budget 放大（1.3×=6.5），且"让他想起某人"的边 cost×0.5
  → "那个花店姑娘...让我想起一个人。希望她照顾好自己。"
  → 这是在特定情绪下才能触发的路径

双向都弱 = 几乎互不主动想起。但一旦有一条边在特定条件下走通了，
那条边会被强化 +0.02，下次稍微容易一点。
```

## 同类型节点互相渗透（模糊记忆全景）

游戏第 60 天，鹿岛樱记忆图的核心区域。精确锚点大量衰减，
同类节点之间通过 similar_to 边形成模糊记忆团：

```
═══════════════════════════════════════════════════════════
同类时间 → 时间团
═══════════════════════════════════════════════════════════

  秋天 ──similar_to 0.7──→ 上个月
    │                        │
    │similar_to 0.5          │similar_to 0.5
    │                        │
  冬天 ←─────────────────────┘
  (它们共同指向几个"经常吃火锅"事件 → 向量越来越近)

  结果: "去年秋冬...还是今年年初？那阵子吧。"
        → 时间变成一个模糊的区间而不是一个点

═══════════════════════════════════════════════════════════
同类情绪 → 情绪团
═══════════════════════════════════════════════════════════

  感激 ──similar_to 0.85──→ 温暖
    │                         │
    │similar_to 0.6           │similar_to 0.7
    │                         │
  安心 ←──────────────────────┘
    │
    │similar_to 0.4（较弱：不同大类）
    │
  担心 ←──→ 低落（另一个负面情绪团）

  同样的事件团（帮过忙的那些事）反复激活同几个正面情绪
  → 它们互相渗透成 "正面的感觉" 这个模糊情绪区

  结果: "对他有一种...感激吧，也不对。
        更像是温暖的感觉。说不上来。
        反正——跟他待在一起很安心。"
        → 说不出精确情绪，但在模糊情绪团里感觉到了方向

═══════════════════════════════════════════════════════════
同类地点 → 地点团  
═══════════════════════════════════════════════════════════

  花店 ──similar_to 0.5──→ 面包店
    │                        │
    │similar_to 0.6          │similar_to 0.4
    │                        │
  喫茶店 ←───────────────────┘
    │
    │similar_to 0.3
    │
  河边（远一点，单独成团）

  结果: "那天是来花店还是面包店...反正在街上。
        千早也在。还是说是小李？"
        → 具体地点模糊了，但"商店街"这个区域感觉清晰

═══════════════════════════════════════════════════════════
同类物品 → 物品团
═══════════════════════════════════════════════════════════

  毛肚 ──similar_to 0.65──→ 牛肉
    │                         │
    │similar_to 0.5           │similar_to 0.5
    │                         │
  豆腐 ───────────────────────┘
    │
    │similar_to 0.4
    │
  可乐（另一类：饮品团）

  结果: "点了好多菜...毛肚牛肉豆腐什么的。
        每次都点差不多的那几样。"

═══════════════════════════════════════════════════════════
同类人物 → 人物团（通过 similar_to，不同于 relationship）
═══════════════════════════════════════════════════════════

  和叶 ──similar_to 0.4──→ 龙之介
          "都是安静的人"
          
  和叶 ──similar_to 0.3──→ 九条
          "都喜欢深夜一个人待着"

  注意：这是 similar_to（"他们很像"），
        不是 relationship（"我跟他们有交情"）。
        两种边在图中并行。

═══════════════════════════════════════════════════════════
检索时的效果：精确检索 vs 模糊扩散
═══════════════════════════════════════════════════════════

新鲜记忆（精确）:
  查"毛肚" → 只有 involved 和 contains 边
  → "千早点了毛肚"
  → 精确、确定

久远记忆（模糊）:
  查"毛肚" → involved/contains + similar_to 边
  → similar_to 指向牛肉、豆腐
  → "点了毛肚...还是牛肉来着。反正就那些火锅菜"
  → 不精确，在同类节点团里扩散

  similar_to 边在检索时被视为"低优先路径"
  → 先走强边（involved/felt → 直达人物和情绪）
  → 强边走完了才开始走 similar_to
  → 所以 NPC 先说出确定的内容，再模糊地补充
  → "千早点了毛肚（确定）。好像还点了牛肉（模糊扩散）"
```
