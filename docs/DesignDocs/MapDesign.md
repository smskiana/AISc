## 1. 设计理念

### 1.1 核心原则

| 原则 | 说明 |
|------|------|
| **可步行的一条街** | 200m 老街，从西到东步行约 2-3 分钟（游戏时间 15 分钟） |
| **每家店都有"里面"** | 店铺不是背景板，玩家可以走进每一家店 |
| **河边是灵魂** | 樱花道 + 河流是最重要的景观，也是情感事件的发生地 |
| **拱廊下的日常** | 商店街有拱廊顶棚，雨天也能在街上走 |
| **左邻右里** | 各家店铺的方位关系影响 NPC 的日常动线 |

### 1.2 风格参考

- **玉子市场** — 商店街的整体氛围、拱廊、兔山商店街
- **夏目友人帐** — 乡间小路、河边、老房子的质感
- **飞翔的魔女** — 小镇的宁静感、日常的节奏
- **深夜食堂** — 店铺内部的温暖感、小空间的利用

### 1.3 2D 视角选择

```
推荐：侧俯视 2.5D（类似星露谷物语 / 牧场物语）

  ┌────────────────────────────────┐
  │ 天空 / 远景                     │
  │    ⛅ 云                        │
  │        🌸 樱花树（远景）         │
  │  ┌──────────────────────────┐  │
  │  │ 河流（中景）              │  │
  │  │ ~~~~水流~~~~              │  │
  │  ├──────────────────────────┤  │
  │  │ 河岸 / 樱花道             │  │
  │  │ 🏠🏠🏠🏠 商店街店铺（前景）│  │
  │  │ 🚶 玩家活动区域           │  │
  │  └──────────────────────────┘  │
  └────────────────────────────────┘

实际 Unity 实现：纯 2D Tilemap，摄像机从侧面略高的角度俯视
街道横向展开（X 轴为主），纵向为深度（Y 轴，近→远）
```

## 3. 商店街主街详细设计

### 3.1 主街分段

主街全长约 200 米，在 Tilemap 中约占 **400 格**（1 格 ≈ 0.5m）。

```
← 西（入口）                                                    东（尽头）→

[0-50]        [50-120]     [120-200]     [200-280]    [280-350]    [350-400]
  入口段        面包店段      中央段        书店段       和果子段      派出所段
  ┌────┐      ┌────┐       ┌────┐       ┌────┐       ┌────┐       ┌────┐
  │牌坊│      │小麦│       │花時│       │猫之│       │龙月│       │交番│
  │    │      │色  │       │計  │       │书架│       │堂  │       │    │
  └────┘      │    │       │    │       │    │       │    │       └────┘
              └────┘       └────┘       └────┘       └────┘
  [公告栏]    [面包香气]   [花摆在]     [旧书摊]     [和果子]     [猫聚集地]
  [贩卖机]                 [门口]                     [橱窗]
                              │
                         喫茶店(玩家)
                         主街中央段·南侧
```

### 3.2 主街 Tilemap 分层结构

```
主街 Tilemap（从下到上，Y 轴 = 深度）：

Layer 6 — 天空背景（天空渐变、远山剪影）
Layer 5 — 远景樱花树（河对岸的树，模糊/虚化）
Layer 4 — 河流层（河水、桥、河岸的树）
Layer 3 — 建筑后层（店铺的屋顶、招牌、二楼窗户）
Layer 2 — 拱廊顶棚（半透明，可 toggle 显示/隐藏）
Layer 1 — 建筑前层（店铺门面、墙面、橱窗、入口）
Layer 0 — 地面层（石板路、拱廊地面、街道纹理）
         ↑ 玩家和 NPC 行走在这一层
```

### 3.3 主街各段点位明细

#### 段 1：入口段 (street.entrance_area) — X: 0~50

```
点位:
  street.entrance_arch      — 入口牌坊「樱桥通」(X:5, Y:6)
  street.bulletin_board     — 公告栏 (X:25, Y:12)
  street.vending_machine    — 自动贩卖机 (X:30, Y:12)
  street.arcade_start       — 拱廊起点 (X:15)
```

**视觉要点**：
- 牌坊是木制鸟居风格，挂着手写的「樱桥通」木匾
- 公告栏贴着社区通知、手绘海报、猫咪走失启事
- 贩卖机有咖啡、果汁、热玉米浓汤（冬天）

#### 段 2：面包店段 (street.bakery_area) — X: 50~120

```
点位:
  bakery.doorway            — 店门 (X:85, Y:12) ← 南侧
  street.bakery_display     — 门口小黑板，写着今日推荐 (X:82, Y:12)
  street.bakery_bench       — 店门口小长椅 (X:92, Y:12)
```

**视觉要点**：
- 面包店是南侧第一家店，暖黄色门面
- 门口小黑板上千早用彩色粉笔写着"今日推荐：蜜瓜包！🐰"
- 上午 9 点后店门打开，传出面包香气（粒子效果？）
- 偶尔千早会抱着纸袋冲出来送面包

#### 段 3：中央段 (street.central_area) — X: 120~200

```
点位:
  flower_shop.doorway       — 花店门 (X:135, Y:4) ← 北侧
  street.flower_stand       — 门口花架 (X:132, Y:12) ← 摆在外面的花
  player_cafe.doorway       — 喫茶店门 (X:160, Y:12) ← 南侧
  street.cafe_sign          — 喫茶店招牌 (X:158, Y:4)
  street.cafe_outdoor       — 喫茶店门口露天座（2 张小桌）(X:168, Y:12)
```

**视觉要点**：
- 花店在北侧，门面爬满藤蔓，橱窗里摆着当季花卉
- 门口花架上每天早上会多一枝新花（鹿岛樱放的）
- 喫茶店是南侧中央，木质老门，玻璃窗能看到里面的吧台
- 店名未定，招牌是奶奶留下的老式木雕
- 露天座有两张小圆桌，晴天可以坐在外面

#### 段 4：书店段 (street.bookstore_area) — X: 200~280

```
点位:
  bookstore.doorway         — 书店门 (X:240, Y:4) ← 北侧
  street.book_cart          — 门口旧书推车（100 円专区）(X:237, Y:12)
  street.bookstore_window   — 大橱窗，能看到里面的旧沙发 (X:245, Y:4)
```

**视觉要点**：
- 旧书店门面是深棕色木头 + 绿色遮阳棚
- 门口有个带轮子的旧书推车，写着"100 円"
- 橱窗里能看到一张旧沙发和堆到天花板的书架
- 店名「猫之书架」的招牌上画着一只打盹的猫

#### 段 5：和果子段 (street.wagashi_area) — X: 280~350

```
点位:
  wagashi.doorway           — 和果子店门 (X:315, Y:12) ← 南侧
  street.wagashi_display    — 橱窗展示柜 (X:310, Y:12) ← 精美的和果子模型
```

**视觉要点**：
- 和果子店门面是传统的日式木质格子窗
- 橱窗里摆着精美的和果子模型（龙之介爸爸的手艺）
- 店门经常半掩着，不像其他店那样敞开
- 二楼的窗户拉着窗帘（龙之介的房间）

#### 段 6：派出所段 (street.police_area) — X: 350~400

```
点位:
  police_box.doorway        — 派出所门 (X:375, Y:4) ← 北侧
  police_box.bench_outside  — 派出所门口长椅 (X:372, Y:12)
  street.cat_corner         — 猫的聚集角 (X:380, Y:12) ← 几个猫碗
  street.east_end           — 街道东尽头，通向公园的小路 (X:395, Y:8)
```

**视觉要点**：
- 派出所是最小的建筑，灰白色外墙 + 蓝色警灯
- 门口长椅上经常趴着一两只猫
- 角落里放着九条放的猫碗和水盆
- 东尽头是一条上坡的小路，通向河边公园
- 站在这里回头看，整条商店街尽收眼底

## 7. Unity Tilemap 技术规格

### 7.1 基础参数

```
项目              值                  说明
─────────────────────────────────────────────
Tile 尺寸         32 × 32 px          每个 tile 32 像素
Pixels Per Unit   32                  1 unit = 1 tile
地图比例          1 tile ≈ 0.5m       200m ≈ 400 tiles
主街 Tilemap      400 × 30 tiles      主街最长最宽的区域
每个店铺内部      16-20 × 10-14 tiles 独立 Tilemap 或同一 Grid 的偏移区域
公园              30 × 20 tiles
河边              40 × 12 tiles
Grid 类型         Rectangle           Unity Tilemap 默认
Sorting Order     见分层表
```

### 7.2 Tilemap 分层方案

```
GameObject 结构:

Grid (MainGrid)
├── Background (Sorting Layer: "Background", Order: 0)
│   ├── Sky (天空渐变·远景)
│   └── FarTrees (远景树·剪影)
│
├── Midground (Sorting Layer: "Midground", Order: 1)
│   ├── River (河流·水面动画)
│   ├── RiverBank (河岸·桥)
│   └── CherryTrees (河边樱花树)
│
├── Buildings_Back (Sorting Layer: "Buildings_Back", Order: 2)
│   ├── Roofs (店铺屋顶)
│   ├── Signs (招牌·霓虹灯)
│   └── SecondFloor (二楼窗户)
│
├── Arcade (Sorting Layer: "Arcade", Order: 3)
│   └── ArcadeRoof (拱廊顶棚·半透明)
│
├── Buildings_Front (Sorting Layer: "Buildings_Front", Order: 4)
│   ├── Walls (墙面)
│   ├── Windows (窗户·橱窗)
│   ├── Doors (门)
│   └── Decorations (装饰物·花架·招牌)
│
├── Ground (Sorting Layer: "Ground", Order: 5)
│   ├── Street (石板路)
│   ├── Sidewalk (人行道)
│   └── Details (地面细节·井盖·落叶)
│
├── Objects (Sorting Layer: "Objects", Order: 6)
│   ├── Furniture (长椅·贩卖机·公告栏)
│   ├── Plants (花盆·杂草)
│   └── Props (猫碗·书推车)
│
└── Foreground (Sorting Layer: "Foreground", Order: 7)
    ├── Overhead (头顶遮挡·电线·灯笼)
    └── Weather (雨·雪粒子效果层)
```

### 7.3 碰撞与导航

```
碰撞体类型:
  • Tilemap Collider 2D + Composite Collider 2D
    — 建筑墙壁、不可行走区域
  • 单独 BoxCollider2D
    — 柜台、展示柜等可交互但不可穿过的物体

NavMesh / 寻路:
  • 使用 Unity 2D NavMesh 或 A* Pathfinding
  • Walkable 层: Ground layer
  • Obstacle 层: Buildings_Front, Objects
  • NPC 路径沿主街 X 轴为主，支路通向公园/河边

玩家碰撞:
  • 玩家有 CapsuleCollider2D 或 BoxCollider2D
  • 与建筑碰撞体交互（不可穿墙）
  • 与门触发区域交互（触发场景切换或店门打开动画）
```

### 7.4 店铺内部实现方式

```
方案 A（推荐）: 同一场景，店铺内部在 Tilemap 的其他位置
  优点: 无缝，不需要异步加载
  缺点: 店铺内部需要放在主街 Tilemap 的旁边（偏移），摄像机切换

方案 B: 每个店铺内部为独立场景
  优点: 每个店铺独立设计，不受主街 Tilemap 范围限制
  缺点: 场景加载有轻微延迟，需要管理场景切换

  → 建议: 方案 A（主街 + 店铺内部在一个大 Tilemap 中）
  
  Tilemap 总尺寸: 约 600 × 50 tiles
  ┌──────────────────────────────────────────────┐
  │  (Y:30-50)  河边 + 公园                      │
  │  (Y:14-30)  主街                             │
  │  (Y:0-14)   各店铺内部（横向排列在主街下方）   │
  │   [bakery] [flower] [cafe] [book] [wagashi] [police] │
  └──────────────────────────────────────────────┘
  
  摄像机按区域 clamp，切换到店铺内部时摄像机 Y 轴移动到下方
```

## 8. 场景分层与渲染顺序

### 8.1 渲染层级总结

```
Y 轴（深度）= Sorting Order 的判定依据

玩家/NPC 的 SpriteRenderer:
  Sorting Layer = "Characters"
  Order in Layer = 基于 Y 位置动态计算 (transform.position.y * -10)
  
  越靠北（Y 越大）= 越远 = 越早渲染（在下方图层）
  越靠南（Y 越小）= 越近 = 越晚渲染（在上方图层）
```

### 8.2 摄像机设置

```
主摄像机:
  • 正交投影 (Orthographic)
  • Size: 约 8-10（显示约 16-20 格高度）
  • 跟随玩家 X 轴移动（左右卷轴）
  • Y 轴可微调（公园区域略向上，商店街居中）
  • 平滑跟随 (Cinemachine 或手写 smooth damp)

区域切换:
  • 玩家走进店铺门 → 摄像机 Y 轴平滑下移 → 显示店铺内部
  • 玩家从主街东端走向公园 → 摄像机 X/Y 平滑移动到公园区域
  • 切换时间: ~0.5 秒（快速但不生硬）
```
