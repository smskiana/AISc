### NPC 导航反直觉修正

2026-07-12 针对 Play 模式中观察到的“虚空移动 / 隔空交流”补了一轮收口：

1. `AStarMovementProvider` 会识别传送 link 路径段，并直接把 NPC 放到出口可走格中心，不再把传送入口到出口当作普通线段走。
2. `AStarMovementProvider.MoveToLocation()` 会在寻路前检查 NPC 当前格；若当前格不可走，会先吸附到最近可走格中心，避免 A* 起点修正被动画播放成“直线走向传送点”。
3. `AStarPathfinder.SimplifyCollinearCells()` 会保留非相邻图边两端点，避免传送入口 / 出口被路径简化吞掉。
4. `AStarMovementProvider` 新增 `[AStarMove:npc_id/name]` 导航诊断日志，输出请求、候选、路径摘要、传送段、长路径段和完成位置。
5. Play 模式曾抓到千早从 `(-1.162,0.749)` 直线走向 `(-11.462,-0.451)`，终点贴近花店传送出口但起点离街道入口约 `0.62`，旧容差约 `0.50`，导致未识别为传送。
6. 传送入口识别容差已放宽到 `triggerRadius + cellSize * 4`；仍要求 waypoint 靠近传送出口，降低普通长段误判风险。
7. `NpcBubbleManager` 收到 `NPC_SOCIAL_ACTION` 后会驱动双方先移动到社交地点。
8. NPC-NPC 气泡显示前会等待双方物理距离接近；超时则跳过该条气泡，避免隔着店铺或地图块显示对话。
9. 若后续仍出现非传送段虚空行走，优先检查导航烘焙的 Sprite bounds 是否把视觉空白当作可走区域。

## 当前已知问题与风险

### 1. 打包 Player 未实测

本轮已验证 Unity 编辑器 Play 模式，但没有打包 Windows Player 做退出流程验证。

后续如果进入发行前联调，建议补一次独立 Player：

1. 启动前确认 8766 无监听。
2. 启动 Player，确认自动拉起后端。
3. 关闭 Player，确认后端进程退出。

### 2. 后端进程树风险

当前 `backend/run.py` 直接执行 `uvicorn.run()`，记录到的 Python 进程就是服务进程。

如果未来后端入口改为派生独立子进程，需要把 `ShutdownOwnedBackend()` 升级为进程树关闭。

### 3. 旧 2026-07-11 风险仍存在

继续保留：

1. 午夜性能偏重。
2. 日计划 LLM 仍可能产出非法空 location。
3. 旧 v0.5 设计文本仍大量保留，判断近期实现时先看 2026-07-11 的 v0.7 追加口径。

### 4. 对话感知层未做真实 LLM 长测

本轮已完成配置、编译、规范检查和 Prompt smoke，但还没有启动真实 LLM / Unity Play 模式观察长时间对白质量。

后续建议重点看：

1. LLM 是否稳定使用现场物件，而不是凭空增加不存在的物品。
2. 和叶、龙之介、九条是否能仅凭现场感知和视觉标签保持自然称呼。
3. `# 现场感知` 是否过长；如果后续发现响应变慢，可进一步压缩物品和 spot 输出。

### 5. Unity 像素地图第二批仍未接入场景

`docs/AIChanges/FrontendArchitecture/2026-07-12_Unity像素地图第二批补图_execution.md` 已记录图片生成结果，但资源仍只是整图 PNG：

1. 尚未通过 Unity MCP 接到场景 / Prefab。
2. 尚未配置 Collider、DoorTrigger、Spot Anchor。
3. 旧书店与和果子店存在少量伪文字 / 图标化细节，正式发行前建议再做手修或重出。

### 6. A* 寻路仍需 Play 模式验证

当前 A*、Sprite 组合烘焙能力、SceneAnchor 区域候选点和 `NavGrid_Main.asset` 传送 links 已完成脚本 / 资产层基础闭环；`gameManager_st` 上的 `NpcSpawner._navigationGrid` 已指向 `NavGrid_Main.asset`。仍需要继续 Play 模式验证路径表现，尤其是非传送段是否还会因为 Sprite bounds 过大而走到视觉空白区。

后续接手时建议：

1. 为关键 `SceneAnchor` 配置圆形 / 方形可站区域。
2. 用 `NpcNavigationDebugController` 控制 `sakura` 到 `player_cafe` / `street` 等地点，观察实际路径和传送 link 是否被使用。
3. 若仍有非传送段虚空行走，检查地图 Sprite bounds、障碍物列表和 air wall，必要时升级为 alpha 采样或显式可走区域烘焙。
