# 会话交接文档 — 2026-07-11

## 当前状态总览

### 记忆系统当前口径

2026-07-11 的多轮执行后，记忆系统已经从早期 v0.5 “完整联想记忆图”收敛为当前 v0.7 工程口径：

1. **轻量图 + 向量数据层**
   - SQLite 图层存节点 ID、边、边类型、双向 clarity。
   - LanceDB / SQLite 向量层存节点内容、embedding、importance、created_day、archived。
   - 图负责联想路径，向量层负责语义内容和兜底。

2. **三种检索模式已落地**
   - `player_dialogue`：玩家对话，最多 2 跳，偏玩家背景和商店街旧事。
   - `npc_dialogue`：NPC 闲聊，最多 1 跳，偏轻量日常。
   - `nightly_impression`：夜间印象刷新，最多 2 跳，比白天更宽。

3. **玩家背景已长期记忆化**
   - Day 0 玩家背景 / 喫茶店 / 奶奶 / 樱桥通熟人社会已写入每个 NPC 的图和向量层。
   - 不再依赖 prompt 硬编码事实白名单。

4. **时间语义已补齐**
   - 检索输出带 `[Day 0]` / `[第N天]` 标签。
   - Day 0 传闻用“听说 / 好像听说”表达。
   - 后续玩家真实行动应写入 Day N 新记忆，自然更新旧语义。

5. **事实守门已收窄**
   - `[FACT_GUARD]` 只拦“亲眼看见 / 自己帮忙 / 玩家正在做某事”这类未发生玩家行动断言。
   - 传闻式表达交给时间语义和记忆系统处理。

6. **可信度增强已补一轮**
   - 后端已支持 `PLAYER_EVENT`，玩家真实行动可写入 NPC 短期记忆，午夜自然入图。
   - 融合节点已记录 `memory_merge_sources` 来源追溯。
   - 检索会写 `memory_retrieval_logs`，记录图命中、向量兜底、LLM 路由、本地跳过和融合命中。
   - `/api/npc/{npc_id}/graph?detail=true` 可返回节点、边、融合来源和最近检索诊断。

## 当前已知问题与风险

### 1. 午夜性能偏重

30 分钟后端独立长测中，午夜平均耗时约 37 秒，最大约 46 秒。

后续如果继续优化记忆系统，优先考虑：

1. 白天检索减少每跳 LLM 路由，更多使用本地排序。
2. 夜间印象检索结果缓存或批量节点读取。
3. 行为计划重复触发去重，减少无意义短期记忆。

### 2. 节点融合已有基础闭环，但仍不是完整退化主机制

当前 `EvolutionEngine` 已补齐基础边继承：

1. 融合节点会继承旧节点邻边。
2. 旧节点会从 SQLite 图层退场，并在向量层标记 archived。
3. 旧节点内部边不会继承，避免自环和无意义路径。

但它仍不是旧 v0.5 的完整四阶段退化主机制：

1. 不要继续围绕融合堆复杂反思 / precision / similar_to 全图语义。
2. 后续若继续加强融合，优先补来源追溯表、长测可达性统计和融合质量评估。
3. 当前主遗忘机制仍是边 clarity 衰减 + 孤点 archived 归档。

### 3. 旧 v0.5 设计仍有历史文本

`MemoryArchitecture.md` 和 `graph-memory-model.md` 中仍保留大量 v0.5 远期设想。

后续判断优先级：

1. 先看 2026-07-11 追加口径。
2. 再看旧章节作为设计背景。
3. 不要把 `similar_to` 全图建边、完整四阶段退化、节点 energy/precision 当作近期必须补齐项。

### 4. 日计划仍有非法空地点风险

后端 30 分钟独立跑测发现 LLM 日计划可能产生空 location，后端会过滤但会导致计划缺块。

建议后续单独处理：

1. 强化日计划 prompt 的固定 ID 约束。
2. 解析层对缺 location 的计划做 routine 兜底。
3. summary 单独统计非法计划数量。
