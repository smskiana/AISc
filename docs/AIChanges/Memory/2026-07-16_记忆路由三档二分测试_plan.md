# 记忆路由三档二分测试 - 设计方案

## 需求理解

基于当前多跳图检索的本地自信路由与 LLM 路由，使用真实项目记忆图和真实 LLM 测出三个玩家可选档位：性能、平衡、质量。

## 当前口径

1. 所属 Workstream：`docs/Workstreams/Memory/README.md`。
2. 沿用 ADR-0001 的 v0.7 轻量图记忆边界，不改变图、向量兜底或对话注入流程。
3. 本轮只评估并产出路由阈值，不直接修改正式运行阈值。

## 测试方案

1. 复制 `backend/data/game.db` 与 LanceDB 到隔离产物目录，不修改正式存档。
2. 从真实图中按 `player_dialogue`、`npc_dialogue`、`nightly_impression` 采集需要在多个候选中选边的路由样本。
3. 每个样本调用真实 LLM 三次，以多数选择形成稳定基线。
4. 将 `local_route_min_score` 与 `local_route_margin` 映射到同一条单调激进度轴：激进度越高，越容易使用本地 Top N。
5. 分别对三种模式和三个档位进行二分搜索：
   - 质量：禁用本地自信跳过，始终使用 LLM 路由，效果保持率保持 100%。
   - 平衡：效果保持率不低于 95%。
   - 性能：效果保持率不低于 75%；首轮实测显示当前小图的下一档质量台阶会由 100% 直接降到 75%，因此用该边界形成可区分选项。
6. 输出每档的阈值、LLM 节省率、本地接管样本数、选择重合度、真实 LLM 平均延迟与延迟估算。
7. 支持显式指定 SQLite、LanceDB、游戏时间与玩家位置；Day 4 复测使用 `memory_route_day4_current_source`，不再读取正在运行的正式数据库。

## 涉及文件

1. `backend/scripts/tune_memory_route_profiles.py`：隔离采样、真实 LLM 基线、二分搜索和报告生成。
2. `backend/tests/test_memory_route_profiles.py`：激进度映射、质量计算和二分边界单测。
3. `docs/AIChanges/artifacts/memory_route_profiles/`：原始 JSON 与玩家可读 Markdown 报告。
4. 本 plan 与对应 execution。
5. `docs/Workstreams/Memory/README.md`：回写经过实测的三档建议入口。

## 风险与控制

1. 真实 LLM 有随机性：每个样本三次、低温调用，并使用多数选择。
2. 当前图规模较小：报告必须披露各模式有效样本数，不把小样本结论包装成永久参数。
3. 同模型既作为路由基线又用于线上路由：本轮衡量的是“相对始终调用当前 LLM 的效果保持率”，不是绝对叙事质量。
4. API 调用成本：限制每模式样本数，并缓存原始响应到产物目录。
5. `aisc_debug`：本轮是隔离后端检索测试，不启动 Unity 运行态，因此该诊断钩子不适用；执行记录中明确说明。

3. 使用隔离数据运行真实 LLM 测试。
4. 检查正式数据库和运行配置未被修改。
