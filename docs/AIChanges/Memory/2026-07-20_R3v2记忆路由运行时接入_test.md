# R3 v2 记忆路由运行时接入 - 独立测试记录

## 1. 结论

初次测试未通过；2026-07-21 修复后有界复测已通过完整验收。初测失败证据保留在第 1-7 节，当前结论以第 8 节为准。

冻结资产、12 条 direct evaluator、正式 worker 成功路径、常驻性能、聚焦自动化、九组合回归和已有故障矩阵通过。真实 worker 12 次请求均采用 `r3_v2`，每次模型调用 1，p95 低于冻结 R3 v2 独立 p95 的 120% 上限，并且关闭后 GPU 占用回落。

初次测试时有两个阻断项：

1. 仓库只有 `evaluate_route_runtime.py` 及 fake probe 自测，没有经批准的隔离 SQLite/LanceDB factory 和带 `expected_node_ids` / `forbidden_node_ids` 的权限 corpus；因此无法对 `r3_v2 / general_llm / local`、最终节点命中、越权、clarity 无副作用和端到端检索墙钟给出真实隔离图证据。
2. 真实禁用 CUDA 时，worker 仍在约 32.26 秒后宣告 READY，首次推理等到 16 秒 timeout 后才以 `specialist_timeout` 回退 local。最终检索方向可收口，但没有做到 CUDA 不可用的早期识别和快速降级。

上述问题已在 execution 第 6 节的独立执行会话修复，并由本记录第 8 节复测通过。

### 8.3 故障修复复测

1. `CUDA_VISIBLE_DEVICES=-1`：真实 worker 在 2107.464 ms 返回 `specialist_load_failed`，未进入 READY；chain 立即采用 local，fallback reason 为 `specialist_unavailable`，模型调用 0，关闭后为 `closed`。
