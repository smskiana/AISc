## 关键实现说明

### 1. 玩家背景不再只是 Prompt 临时注入

现在每个 NPC 冷启动时都会写入一批共享初始长期记忆，包括：

1. 小李小时候在樱桥通长大，后来去了城市又回来
2. 小李要重开奶奶留下的喫茶店
3. 奶奶去世后喫茶店关了很久
4. 樱桥通是熟人社会，街上消息传得快

这些记忆：

1. 会写进 `memory_nodes`
2. 会编码后写进 LanceDB
3. 会与 `self` 和 `player person` 建立 `involved` 结构边

这样 NPC 对玩家说话时，图检索和向量检索都能从长期记忆里直接捞到这些真相，而不是只靠 Prompt 补丁。

### 2. 玩家分支检索改成“更容易先想到玩家相关记忆”

本次没有压低 NPC 私人核心记忆，只做正向增强：

1. `target_id == "player"` 时，向量查询词会额外带上：
   - `喫茶店`
   - `奶奶`
   - `从城市回来`
   - `樱桥通`
2. 节点重排时，命中这些关键词会获得额外加分
3. 玩家人物节点出发的图候选仍然保留 `source_is_target` 优势

结果是：

1. 玩家背景会更靠前
2. 九条莲自己的核心旧事仍然还能被回忆出来，不会被硬压掉

### 3. 记忆库已做全量重置

这次不是只删 SQLite。

已执行：

1. `memory_nodes / memory_edges / short_term_memories / npc_impressions / player_memories` 清空后重建
2. LanceDB 全表清空
3. 冷启动重新灌入初始长期记忆

这样旧的错误记忆，比如：

1. “玩家开的是面包店”
2. “奶奶开的是和果子店”
3. “红豆大福的味道”

都不会继续残留在新的记忆库里。
