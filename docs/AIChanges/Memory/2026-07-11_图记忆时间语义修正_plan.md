# 图记忆时间语义修正 — 设计方案

## 问题

上一版 `PLAYER_FACT_BOUNDARY` 把玩家背景写成了静态 prompt 白名单，这会让 Day 0 的状态在后期仍被当成当前事实。

更合理的方向是：

1. 初始背景本身就是图/向量记忆的一部分。
2. 初始背景必须带时间和语气，例如 `Day 0 听说...`。
3. 检索结果进入 prompt 时必须保留时间标签。
4. 后续玩家行为产生更新记忆后，LLM 能根据时间先后判断“旧背景”和“新事实”的关系。

## 修复方案

### 1. 取消硬编码玩家事实白名单

移除“关于玩家只能确定以下事实”的写死提示，改成通用时间规则：

1. 优先按记忆时间判断先后。
2. 旧记忆只代表当时知道的事。
3. 如果新记忆更新了旧说法，以新记忆为准。
4. 不要把“听说/准备/可能”改写成亲眼事实。

### 2. 初始共享记忆改成 Day 0 语气

把：

```text
小李准备重新开奶奶留下的喫茶店
```

改成：

```text
Day 0 前后，街上听说小李准备重新开奶奶留下的喫茶店。
```

这样它天然是“初始传闻/意图”，不是当前永恒状态。

### 3. 检索输出保留时间标签

现在向量节点已有 `created_day`，SQLite 图节点也有 `created_at_game_time`。本轮先用向量层 `created_day` 输出：

```text
- [Day 0] 听说: ...
- [第3天] 记得: ...
```

这样 LLM 拿到两条有先后顺序的记忆时，可以组织上下文。

### 4. 保留轻量输出守门但收窄含义

`[FACT_GUARD]` 暂时保留，只防止 LLM 把没有时间证据的“玩家已做行为”写进短期记忆。后续当玩家行为也进入图记忆后，再升级为“查图后放行”。
