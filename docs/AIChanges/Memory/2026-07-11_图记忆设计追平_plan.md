# 图记忆设计追平 — 方案

## 需求理解

根据 2026-07-11 多份执行记录，当前记忆系统实现已经从早期 v0.5 “完整联想记忆图”收敛为更轻的 v0.6/v0.7 工程口径：

1. SQLite 图层主要存节点 ID、边、边类型与双向 clarity。
2. LanceDB / SQLite 向量兜底层存节点内容、向量、importance、created_day、archived。
3. 图是联想索引层，不是全部记忆内容的唯一存储层。
4. 白天玩家对话、NPC 闲聊、夜间印象刷新已经拆成不同检索模式。
5. 向量检索不再作为白天主检索，只在图结果不足时少量兜底。
6. Day 0 玩家背景已经进入长期图/向量记忆，并用时间标签约束事实语义。

本轮目标是把这些执行口径补回设计文档，避免后续继续按旧 v0.5 的 full graph / similar_to / precision / energy 方案推进。

## 设计追平边界

本轮只更新设计与交接文档，不修改后端代码。

保留旧 v0.5 章节作为历史设计背景，但追加明确优先级：

1. 当前实现优先级高于旧 v0.5 文本。
2. 轻量图 + 向量数据层是近期执行口径。
3. `similar_to` 全图建边、节点 energy/precision 四阶段完整退化、复杂思考预算策略，暂不作为近期必做项。
