# 玩家提示接入向量记忆 — 设计方案

## 需求理解

当前玩家的三条建议回复只基于：

1. 当前几句对话
2. 当前时间地点
3. NPC 当下状态与印象

还没有参考玩家自己的向量记忆。

本次目标是把“玩家关于当前 NPC 的历史记忆”接入建议回复生成链路，让三条提示更像玩家在延续自己之前聊过的话题，同时继续保留自由输入。

## 现状确认

### 已有能力

`PlayerDialogueService` 已经有：

1. 玩家对话结束后的向量记忆写入
2. `search_player_memories(...)` 查询入口

### 当前缺口

`_build_player_reply_choices(...)` 在生成建议回复时还没有调用玩家记忆查询，因此 prompt 中没有“玩家以前和这个 NPC 聊过什么”的上下文。

## 修改原则

1. **最小改动**：只改后端提示生成链路，不改协议、不改前端
2. **失败降级**：查不到记忆时仍按当前上下文生成三条提示
3. **上下文克制**：只取少量高相关玩家记忆，避免 prompt 过重

## 涉及文件

1. `backend/src/application/dialogue_service.py`
2. `backend/src/dialogue/prompt_builder.py`
3. `docs/AIChanges/Memory/2026-07-10_玩家提示接入向量记忆_plan.md`

## 具体方案

### 1. 在建议回复生成前查询玩家向量记忆

在 `PlayerDialogueService._build_player_reply_choices(...)` 中：

1. 判断当前 target 是否为 `player`
2. 按当前 NPC 查询玩家向量记忆
3. 取前 2-3 条高相关结果
4. 将结果传给 `PromptBuilder.build_player_reply_suggestions(...)`

优先使用 NPC 中文名做查询，以便和玩家记忆写入时的内容格式更贴近。

### 2. PromptBuilder 把玩家记忆纳入建议回复 prompt

在 `build_player_reply_suggestions(...)` 中新增参数：

1. `player_memories`

并在 prompt 中增加一段“玩家可能记得的事”：

1. 只提供摘要化的 2-3 条
2. 允许模型自然参考
3. 明确说明“如果不相关就不要硬用”

### 3. 控制记忆文本长度

为了避免 prompt 过长：

1. 单条玩家记忆截断
2. 总条数控制在 3 条以内
3. 保留最近几句对话作为主要上下文，玩家记忆只是辅助

## 风险点

1. 玩家记忆文本中可能混有较长原始对话，需要做截断清洗
2. 向量搜索结果有时不够精准，因此 prompt 里要强调“相关才参考”
3. 如果玩家还没有任何记忆，必须稳定回退到当前无记忆版本

## 验证计划

1. 后端脚本编译通过
2. `_build_player_reply_choices(...)` 已调用玩家记忆查询
3. `build_player_reply_suggestions(...)` 已接收并注入玩家记忆上下文
4. 没有玩家记忆时仍能正常生成三条提示
