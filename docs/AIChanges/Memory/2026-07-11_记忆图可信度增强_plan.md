# 记忆图可信度增强 — 设计方案

## 需求理解

上一轮盘点后，当前记忆图已经能跑通：

1. 初始长期记忆入图。
2. 对话 / 行为短期记忆午夜提取入图。
3. 多跳图路由检索。
4. 边 clarity 衰减。
5. 夜间印象刷新。
6. 基础节点融合与边继承。

但要让原型更可信，还缺几类增强：

1. 玩家真实行动要能成为 Day N 记忆，更新 Day 0 传闻。
2. 融合节点要有来源追溯。
3. 检索要留下质量 / 性能指标。
4. 白天检索不要所有候选冲突都请求 LLM。
5. 图调试接口要能看到节点和边，而不是只有计数。

## 本轮范围

本轮只做“可信度增强”，不扩展旧 v0.5 完整退化系统。

### 1. 玩家事件入图入口

新增后端消息：

```json
{
  "type": "PLAYER_EVENT",
  "event_type": "cafe_progress",
  "content": "小李今天打扫了奶奶留下的喫茶店。",
  "location_id": "player_cafe.main_hall",
  "importance": 0.75,
  "witness_npcs": ["sakura"]
}
```

处理方式：

1. 写入见证 NPC 的 `short_term_memories`。
2. 如果没有 `witness_npcs`，默认写给同 zone 或相邻 zone 的 NPC。
3. 若仍没有 NPC 可见，则写给所有 NPC 一条“街上传闻”版本。
4. 午夜提取会自然把它入图，成为 Day N 新记忆。

### 2. 融合来源追溯表

新增 `memory_merge_sources`：

```text
merged_node_id, source_node_id, npc_id, source_type, source_value,
similarity, created_at_game_time
```

融合时记录 A / B 来源，方便后续调试、UI 展开和质量评估。

### 3. 检索指标表

新增 `memory_retrieval_logs`：

```text
npc_id, target_id, mode, game_time, location,
graph_nodes, vector_fallback, final_nodes,
selected_edges, llm_route_calls, local_route_skips,
hit_merged_count, elapsed_sec
```

用于后续压测统计“检索是否真的走图、是否依赖向量兜底、是否命中融合节点”。

### 4. 本地路由自信跳过 LLM

在 `RetrievalEngine._route_edges_llm()` 前增加本地自信判断：

1. 如果候选已经少于选择数，直接本地返回。
2. 如果 top N 与下一名分差足够大，也直接本地返回。
3. 只有候选接近、冲突明显时才调用 LLM。

### 5. 图调试接口增强

增强 `/api/npc/{npc_id}/graph`：

1. 默认仍返回 count，兼容旧调用。
2. 支持 `?detail=true&limit=50` 返回节点、边、向量层字段、融合来源。

## 风险点

1. 玩家事件写入不能变成硬编码事实库，只写入短期记忆，仍走午夜提取入图。
2. 检索指标写库不能影响对话主链，失败只记 debug。
3. 本地跳过 LLM 不能太激进，否则可能降低联想质量；阈值先保守。
4. 图调试接口要限制 `limit`，避免一次返回太多。
