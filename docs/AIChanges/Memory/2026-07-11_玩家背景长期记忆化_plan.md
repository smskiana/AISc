# 玩家背景长期记忆化 — 设计方案

## 需求理解

当前问题不是九条莲单独“人设跑偏”，而是玩家基础设定没有作为稳定长期记忆进入 NPC 的长期记忆主链，导致：

1. 玩家背景只在首遇 Prompt 中临时注入
2. 玩家对话检索不够偏向“小李 / 玩家相关”的记忆节点
3. 错误对白已经污染短期记忆和熟人印象，后续继续被复用

用户本轮明确要求：

1. `player_background`、游戏背景、NPC 旧有记忆都应进入图和向量库，形成初始长期记忆
2. `retrieval.py` 需要让玩家分支优先命中“小李 / 玩家相关”节点，但不要压低 NPC 其他私人核心记忆
3. 现在直接重置全部记忆库

## 方案思路

### 1. 把“共享真相源”下沉到冷启动图初始化

在 `StateManager.cold_start()` / `_init_graph()` 中，给每个 NPC 追加一批“初始共享长期记忆”节点：

1. 玩家背景
   - 小李从城市回来
   - 小李准备重开奶奶留下的喫茶店
   - 奶奶的喫茶店曾是街上热闹的地方
2. 商店街背景
   - 樱桥通是小而熟的老商店街
   - 街上几家店彼此熟悉，很多事会互相知道
3. NPC 旧有记忆
   - 继续沿用各自 `npc_profiles/*.json` 里的 `core_memories`

这些节点会和“我 / 玩家人物节点”建立结构边，确保：

1. 图检索能从 `self` 或 `player person` 起点直接命中
2. 向量库里也有同样的初始事实，不再只靠 Prompt 临时硬塞

### 2. 玩家分支检索改为“增强命中”，不是“压制其他核心记忆”

保留 NPC 私人核心记忆可被玩家逐步套出这一能力，但让玩家相关节点更容易先被想到：

1. 向量查询词对 `player` 分支补充玩家背景关键词
2. 节点重排对 `player` 分支增加“小李 / 喫茶店 / 奶奶 / 从城市回来 / 商店街熟人关系”等关键词加分
3. 保留现有核心记忆分值，不额外做负向压制

### 3. 记忆库重置必须同时覆盖 SQLite 和 LanceDB

仅删 SQLite 不够，因为 LanceDB 当前是 merge-upsert，旧向量节点可能残留。

因此需要：

1. 给 LanceDB 客户端增加按 NPC 清表能力
2. `cold_start()` 时先清 SQLite，再清 LanceDB，再重新灌入初始化图和向量

## 备注

我同意这次不再继续往 Prompt 里硬塞 `player_background`。  
这类“世界真相”更适合作为初始长期记忆进入图和向量库，让后续对白、检索、夜间印象生成都读同一套真相源。
