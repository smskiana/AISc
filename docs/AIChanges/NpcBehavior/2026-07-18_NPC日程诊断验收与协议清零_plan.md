# NPC 日程诊断验收与协议清零执行方案

> 执行记录: [2026-07-18_NPC日程诊断验收与协议清零_execution.md](2026-07-18_NPC日程诊断验收与协议清零_execution.md)

## 1. 目标与前置条件

本阶段只在前四阶段完成后执行，负责补齐结构化诊断、隔离控制探针、端到端验收、协议清零和文档收口，不再新增业务能力。

## 2. 实施步骤

1. 扩展 `aisc_debug.daily_schedule_snapshot`，并新增 `interaction_replan_snapshot`；聚合后端 owner trace 与 Unity 托管状态，但不暴露模型原文或 embedding。
2. `aisc_debug` 覆盖计划生成/恢复/提交、候选分组、证据、最终校验、fallback、pending、压缩、执行、replan 和稳定失败原因。
3. `aisc_control` 增加白名单隔离探针：固定输入 planner、供应商超时/fallback、跳到 17:00、社交锁 defer、任务失败和迟到 revision。探针必须调用正式 seam，并标明写入范围。
4. 建立小于 30 秒的回归反馈循环，分别覆盖日程幂等、社交权威、三层 planner、Unity 抢占和 replan。
5. 执行完整 Python 测试、Unity EditMode、PlayMode 长链与编译检查；Unity 运行时优先通过 `aisc_debug` 取证。
6. 全仓协议审计并删除无调用方的旧 DTO、router、callback、sender、拒绝 ACK、死代码和误导注释。
7. 回写 NpcBehavior、FrontendArchitecture、GameTime、Dialogue、ProtocolAndIntegration、TestingAndDiagnostics 入口文档；修正原 plan 的 execution 日期链接。
8. 完成重大更新后重建 codebase-memory 索引，并同步 `docs/AIChanges/codebase-memory-mcp_更新.md`。

## 3. 必须覆盖的验收场景

1. 新游戏、CONTINUE、同日重连、后端重启、旧档、跨日和午夜准备。
2. 玩家对话完成、NPC-NPC COMPLETE、取消/无摘要互动和迟到 replan。
3. 晴雨、closed/unavailable/unknown spot、不可达地点、职业身份和基础 need。
4. 供应商正常、解析失败、120 秒超时、迟到返回和多 owner 部分失败。
5. 长任务跨计划点、社交锁、任务失败、17:00 压缩、暂停、恢复和跳时。
6. 社交 decision/content 迟到、任一方移动、玩家抢占、会合失败和断线。

## 4. 协议清零检查表

- 网络 `NPC_BEHAVIOR` 不再进入正式执行器。
- `NPC_TASK_STATUS*`、`NPC_ACTION_RESULT*`、旧时间控制和旧 wake/sleep push 无生产入口。
- Python 无物理社交扫描、会合监督、任务监督、行为 tick 或世界状态直接写入。
- Unity 无后端分钟行为命令依赖。
- 所有跨端 replacement/effect/result 都有 request、operation、revision 和稳定错误语义。

## 5. 验收标准

- 功能本体、诊断、控制探针、测试和入口文档全部同步。
- execution 逐项记录实际验证证据、未完成项和不适用原因。
- 未完成任何硬约束时不得把原总方案标记为完成。

## 6. 风险与停止条件

- PlayMode 或 Unity MCP 不稳定时不得用单元测试替代长链结论；记录阻塞并保持计划未完成。
- 发现协议仍有外部消费者时，先列出迁移路径，不进行破坏性删除。
