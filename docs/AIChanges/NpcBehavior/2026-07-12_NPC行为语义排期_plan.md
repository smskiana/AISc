## 需求理解

当前需要补充一个排期项：优化 NPC 随机乱动，让 NPC 在对的位置做对的事情。

这里的问题不是单纯寻路，而是“行为语义”：

1. NPC 应该按时间、身份、地点、营业状态、社交状态选择合理行动。
2. 后端行为意图不能只给一个粗略地点和动作。
3. 前端落地时要知道该动作应该在哪个 spot / anchor / 可站区域表现。
4. 行为失败需要反馈给后端，不能让 NPC 状态假装完成。

新增 `docs/Workstreams/NpcBehavior/README.md`，专门承接 NPC 日常行为、动作地点匹配、行为结果回报和“随机乱动”治理。

2. “正在进行”增加 NPC 行为语义收敛。
3. “正在计划”增加行为 spot / action affordance 细化。

### 3. 同步相关工作流

更新：

1. `docs/Workstreams/README.md`

## 风险点

1. 不能把“行为语义”误写成“随机挑一个可达点”，否则只是换一种随机乱动。
2. 不应让 LLM 自由生成地点 / 行为名，仍必须走固定 ID 和 action-location 约束。
3. 前端 spot / anchor 属于资产层事实，后续实现时应优先通过 Unity MCP 配置。

## 验证方式

1. 静态检查 NPC 行为工作流存在。
2. 静态检查 Roadmap 中出现“NPC 行为语义收敛”。
3. 静态检查 ChangeIndex 中新增 NPC 行为分组。
4. 本轮不涉及代码编译。
