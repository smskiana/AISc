# NPC名字牌行为后缀 — 设计方案

## 需求理解

当前 NPC 的计划执行虽然已经接入后端 `action_id`，但由于前端还没有对应动作动画，玩家很难从画面判断角色正在做什么。

本次目标是在不改后端协议、不额外增加预制体配置负担的前提下，为 NPC prefab 内的 `txt_nameplate_dy` 名字牌补充行为后缀，例如：

- `鹿岛樱（工作中）`
- `千早（休息中）`
- `九条莲（移动中）`

## 已确认现状

- `Assets/Scripts/NPC/NpcEntity.cs` 当前在 `Initialize()` 中只把 `_nameLabel.text` 设置为 NPC 名字
- `PlayAction(string actionId)` 已经是所有行为状态切换的统一入口，适合作为名字牌刷新挂点
- `Assets/Scripts/NPC/NpcSpawner.cs` 在收到 `OnNpcBehavior` 时会调用 `PlayAction()`，但新生成 NPC 时尚未立刻套用初始 `current_action`
- 后端行为使用稳定 `action_id`，前端可直接按 ID 分类，不需要改 WebSocket 协议

## 方案原则

1. 最小修改：只改 Unity 前端显示逻辑
2. 不把策划显示名硬编码成一大套外部配置读取器，先在前端按 `action_id` 分类出足够易懂的状态文案
3. 名字牌刷新逻辑集中在 `NpcEntity`，避免分散到调用侧

## 计划修改文件

### Unity 脚本

- `Assets/Scripts/NPC/NpcEntity.cs`
- `Assets/Scripts/NPC/NpcSpawner.cs`

### 文档

- `docs/AIChanges/NpcBehavior/2026-07-11_NPC名字牌行为后缀_plan.md`
- `docs/AIChanges/NpcBehavior/2026-07-11_NPC名字牌行为后缀_execution.md`

## 具体方案

### 1. 在 `NpcEntity` 增加名字牌刷新函数

新增两个内部方法：

- `RefreshNameLabel()`
- `GetActionStatusSuffix(string actionId)`

显示规则为：

- 无动作或空动作：只显示角色名
- 移动类动作：`（移动中）`
- `work_*`：`（工作中）`
- 社交类动作：如 `（交流中）`、`（送东西中）`
- 常见休闲类动作：如 `（休息中）`、`（阅读中）`、`（用餐中）`
- 未细分的特殊或反应类动作：给一个通用但可理解的后缀，避免再次变成“看不懂在干嘛”

### 2. 在动作切换时同步刷新名字牌

在以下时机调用 `RefreshNameLabel()`：

- `Initialize()`
- `PlayAction()`

这样名字牌会和当前 `CurrentAction` 保持一致。

### 3. 让新生成 NPC 也立刻显示当前状态

在 `NpcSpawner.SpawnOrUpdateNpc()` 新建 NPC 的流程里，补一行：

- `entity.PlayAction(state.current_action ?? "stand");`

避免刚进场时只能显示名字，必须等下一条行为消息后才看到后缀。

## 风险点

1. 前端分类文案与 `shared/actions.json` 的显示名不是一一逐字对应，而是偏“状态摘要”
2. 若后端未来新增大量新动作，未命中的动作会走通用文案，需要后续按需补充分支

## 验证计划

1. Unity 编译通过，无新增脚本错误
2. NPC 初次生成后，名字牌就带有当前行为后缀
3. 收到新的 `NpcBehavior` 后，名字牌会随着动作变化同步更新
4. 移动中的 NPC 会明确显示 `（移动中）`，缓解无动画时的信息缺失
