# NPC 自研 A* 寻路方案

## 需求理解

为 2D Sprite 项目的 NPC 移动新增第一版自研 A* 寻路能力：

1. 只处理静态障碍物，不处理动态障碍物。
2. 支持由导航遮罩图片生成可走网格。
3. 支持传送点，路径搜索时可从入口格跳到出口格。
4. 支持把导航网格渲染成调试图片 / 场景覆盖图。
5. 支持控制现有 NPC 走 A* 路径。

## 方案思路

新增 `Assets/Scripts/Navigation/`，把寻路职责拆出 `NpcEntity`：

1. `NavigationGridAsset`
   - ScriptableObject。
   - 保存世界原点、格子大小、宽高、导航遮罩 Sprite、可走颜色阈值、对角移动设置和传送点。
   - 运行时从 Sprite 像素生成静态可走网格。

2. `AStarPathfinder`
   - 纯 C# A* 搜索类。
   - 输入世界坐标，输出世界路径点。
   - 支持 4 向 / 8 向移动。
   - 支持禁止斜向穿墙角。
   - 支持传送点邻接边。

3. `AStarMovementProvider`
   - 实现现有 `IMovementProvider`。
   - NPC 收到目标点后先查路径，再按 waypoint 移动。
   - 保留对话 / 气泡移动锁暂停逻辑。
   - 网格未配置或路径失败时回退为直线移动，避免破坏现有联调。

4. `NavigationGridOverlay`
   - 场景调试组件。
   - 用 SpriteRenderer 渲染半透明导航网格纹理。
   - 可走格 / 不可走格 / 传送入口出口用不同颜色。

5. `NpcNavigationDebugController`
   - 场景调试组件。
   - Inspector 指定 NPC ID 与目标 location_id。
   - 热键触发指定 NPC 移动，方便 Play 模式验证。

## 涉及文件

新增：

1. `Assets/Scripts/Navigation/NavigationGridAsset.cs`
2. `Assets/Scripts/Navigation/AStarPathfinder.cs`
3. `Assets/Scripts/Navigation/AStarMovementProvider.cs`
4. `Assets/Scripts/Navigation/NavigationGridOverlay.cs`
5. `Assets/Scripts/Navigation/NpcNavigationDebugController.cs`

修改：

1. `Assets/Scripts/NPC/NpcSpawner.cs`
2. `Assets/Scripts/Index.md`

## 风险点

1. 导航遮罩 Sprite 的 Texture 需要开启 Read/Write，否则运行时不能读取像素；第一版会给出 warning 并回退直线移动。
2. 具体导航网格资产、场景节点、Prefab 引用属于资产层配置，本轮先提供脚本能力，后续应通过 Unity MCP 在编辑器内创建并连线。
3. 第一版不处理动态障碍物和多 NPC 局部避让；如果后续 NPC 数增加，可再加路径重算或软避让层。
4. 传送点第一版按网格邻接边处理，不做传送动画表现。
