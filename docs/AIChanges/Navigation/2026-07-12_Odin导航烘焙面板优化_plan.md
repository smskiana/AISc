# Odin 导航烘焙面板优化方案

## 需求理解

项目已安装 Odin Inspector，希望让 NPC A* 导航网格与 Sprite 组合烘焙流程更容易使用。

本轮重点优化 Inspector 体验：

1. 烘焙器字段按用途分组。
2. 必填项给出清晰提示。
3. 常用操作做成按钮：收集地图 Sprite、收集障碍物 Sprite、Bake。
4. 显示只读统计，方便确认当前会烘多少 Sprite。
5. 导航资产显示烘焙摘要、网格尺寸和传送点配置，不强迫直接展开大数组。

## 方案思路

1. 在 `NavigationGridSpriteBaker` 中接入 Odin：
   - `TitleGroup` / `BoxGroup` 做结构化分区。
   - `Required` / `InfoBox` 标注关键配置。
   - `Button` 做一键收集和 Bake。
   - `ShowInInspector` / `ReadOnly` 显示当前收集统计与预估网格尺寸。

2. 在 `NavigationGridAsset` 中接入 Odin：
   - 分区显示来源、网格、遮罩兼容项、烘焙结果、移动和传送点。
   - 烘焙数组默认折叠，避免 Inspector 太长。
   - 显示 `BakedSourceSummary` 等只读信息。

3. 用 `#if ODIN_INSPECTOR` 包裹 Odin using 和属性。
   - 当前项目已有 `ODIN_INSPECTOR` 宏。
   - 包裹后可减少未来移除 Odin 时的编译风险。

## 风险点

1. Odin 属性只影响 Inspector，不应改变运行时寻路逻辑。
2. 一键收集会覆盖显式列表，需要按钮文案明确。
3. 统计属性如果每帧频繁计算可能有轻微 Editor 开销；当前 Sprite 数量较少，可接受。
