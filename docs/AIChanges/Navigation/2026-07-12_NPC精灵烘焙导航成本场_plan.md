# NPC 精灵烘焙导航成本场方案

## 需求理解

上一版用黑白导航遮罩 Sprite 生成可走格，但实际制作困难。现在改为：

1. 用地图 SpriteRenderer 组合定义导航覆盖范围。
2. 用障碍物 SpriteRenderer 组合定义静态阻挡。
3. 自动生成 `NavigationGridAsset` 的可走格与通行成本。
4. 距离障碍物 Sprite 越近，格子通行成本越高，NPC 会自然绕开狭窄贴边路线。

## 方案思路

1. `NavigationGridAsset` 保留旧遮罩模式作为兼容，但新增烘焙网格模式。
2. 烘焙结果写入资产：
   - 世界原点
   - 宽高
   - cellSize
   - 每格是否可走
   - 每格通行成本
3. `AStarPathfinder` 的移动代价乘以目标格通行成本。
4. 新增 Editor 工具 `NavigationGridSpriteBaker`：
   - 挂在场景节点上。
   - Inspector 填目标 `NavigationGridAsset`。
   - 填地图 SpriteRenderer 列表或父节点。
   - 填障碍物 SpriteRenderer 列表或父节点。
   - 右键菜单执行 Bake。
5. 障碍物附近成本计算：
   - 障碍物 bounds 内格子不可走。
   - 障碍物 bounds 外按最近距离计算惩罚。
   - 距离 0 到 dangerRadius 内成本从 `maxNearObstacleCost` 衰减到 1。

## 风险点

1. 第一版按 SpriteRenderer 的 world bounds 烘焙，不做逐像素 alpha 形状判定，适合矩形/近矩形障碍物；复杂异形障碍后续可加 alpha 采样。
2. `NavigationGridAsset` 的烘焙数据会存进 ScriptableObject，较大地图会增加资产体积。
3. 烘焙工具是 Editor 能力，场景中具体节点挂载和资产连线仍应后续用 Unity MCP 完成。
