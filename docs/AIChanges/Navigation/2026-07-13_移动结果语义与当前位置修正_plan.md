# 移动结果语义与当前位置修正方案

## Workstream

Navigation

## Roadmap item

拆 `IMovementProvider` 的结果语义；修正 `NpcEntity.CurrentLocation` 到达后更新。

## 相关 ADR

`docs/DecisionRecords/ADR-0002-navigation-typed-path.md`：保持 typed path 语义，不把传送特殊判断继续塞回移动执行层。

## 需求理解

1. 给 `IMovementProvider` 增加移动结果语义：成功、失败、取消。
2. 修正 `NpcEntity.CurrentLocation`，移动开始时不再提前写目标 location，只在确认移动成功到达后更新。

## 方案思路

1. 新增 `MovementResult` 枚举：`Succeeded`、`Failed`、`Cancelled`。
2. 将 `IMovementProvider.MoveTo` / `MoveToLocation` 回调升级为 `Action<MovementResult>`。
3. `LerpMovementProvider` 和 `AStarMovementProvider` 保证每次移动只回调一次结果。
4. `Stop(transform)` 取消正在执行的移动，并回调 `MovementResult.Cancelled`。
5. `NpcEntity.MoveToLocation` 在失败或取消时不更新 `CurrentLocation`，只在 `Succeeded` 时更新。

## 涉及文件

1. `Assets/Scripts/NPC/IMovementProvider.cs`
2. `Assets/Scripts/NPC/NpcEntity.cs`
3. `Assets/Scripts/Navigation/AStarMovementProvider.cs`
4. `docs/AIChanges/Navigation/2026-07-13_移动结果语义与当前位置修正_plan.md`
5. `docs/AIChanges/Navigation/2026-07-13_移动结果语义与当前位置修正_execution.md`

## 风险点

1. 旧回调是 `Action`，需要同步更新所有实现与调用点。
2. `Stop()` 可能由新移动开始前调用，必须把旧移动标记为取消，不能误报成功。
3. A* 候选全部不可达时必须返回失败，不能触发到达语义。

## 验证方式

1. `rg` 检查 `Action onComplete` / `onArrived?.Invoke()` 是否残留错误语义。
2. `dotnet build AISc.sln --no-restore`。
3. Unity MCP 刷新脚本并校验相关脚本，读取 Console 错误。
