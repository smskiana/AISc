# 双端存读档事务设计方案

> 执行记录: [execution.md](2026-07-14_双端存读档事务_execution.md)

## 需求理解

接通 Unity 主存档与 Python 记忆检查点，使同一存档槽的两部分通过 `checkpoint_id` 一致保存和加载。

## Workstream / Roadmap

- Workstream: `ProtocolAndSave`
- Roadmap item: 双端保存 / 读档事务
- 相关 ADR: ADR-0006

## 实现方案

1. Python 注册记忆检查点 prepare / commit / abort / load handler。
2. Unity `UnitySaveService` 独立处理 envelope 响应和仓储事务。
3. `GameManager` 只作为现有 UI 的 facade 和事件桥接。
4. 旧完整存档命令保留兼容，但新 UI facade 默认走 Unity 主存档。

## 风险点

1. 存档列表当前仍来自旧 Python 入口，后续需切到 Unity 本地 manifest。
2. 跨进程无法获得真正单文件事务，必须依赖 checkpoint 身份、临时目录和恢复策略。

## 验证

Python 全套测试、Unity MCP 编译、协议 handler 单元测试和实际连接握手。
