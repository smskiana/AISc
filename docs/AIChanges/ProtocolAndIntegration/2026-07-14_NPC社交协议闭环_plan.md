# NPC-NPC 社交协议闭环设计方案

> 执行记录: [execution.md](2026-07-14_NPC社交协议闭环_execution.md)

## 需求理解

当前后端发出社交动作后立即生成并发送气泡，Unity 仅在首条气泡到达后等待双方靠近 12 秒。需要改为后端先准备、Unity 完成会合、后端再生成内容，并在 Unity 播放完成后才写入记忆。

## Workstream / Roadmap

- Workstream: `Dialogue`、`NpcBehavior`、`ProtocolAndSave`
- Roadmap item: NPC 见闻与社交互动前置闭环
- 相关 ADR: ADR-0002 typed path、ADR-0006 协议边界
- 工程口径变化: NPC-NPC 对话必须经过 Unity 现场 READY，记忆只在 COMPLETE 后提交

## 协议状态机

1. 后端发送 `NPC_SOCIAL_PREPARE`，携带 `request_id`、双方 ID、目标 location 和 120 秒 deadline。
2. Unity 让双方移动到社交地点，并按真实距离判断会合。
3. Unity 回传 `NPC_SOCIAL_READY` 或 `NPC_SOCIAL_FAILED`。
4. 后端仅在 READY 后启动 LLM，并发送带同一 `request_id` 的 `NPC_BUBBLE`。
5. Unity 播放完整队列后回传 `NPC_SOCIAL_COMPLETE`；播放或参与者异常回传 FAILED。
6. 后端仅在 COMPLETE 后写短期记忆和互动影响。

## 职责拆分

- `BehaviorEngine`: 检测机会并调用准备回调，不管理等待状态。
- `NpcSocialSessionCoordinator`: 管理 session、超时、READY/FAILED/COMPLETE 和后台生成。
- `NpcDialogueManager`: 生成带 request_id 的内容，返回 transcript；显式提交记忆。
- `NpcSocialProtocolController`: Unity 会合状态与结果回报。
- `NpcBubbleManager`: 队列播放、完成/失败通知，不再使用 12 秒气泡等待兜底。

## 风险点

1. 旧 `NPC_SOCIAL_ACTION` 消息可能仍在飞行，Unity 保留兼容处理但不用于新链路。
2. 玩家对话抢占 NPC-NPC 会话时必须回传失败并释放移动锁。
3. LLM 完成后 Unity 断线，不得写入“已发生”记忆。
4. 同一 NPC 同时参与多个会话必须由后端 coordinator 拒绝。

## 验证

1. Python 测试覆盖 READY 前不调用 LLM、失败不写记忆、COMPLETE 后提交、超时清理和旧请求拒绝。
2. Unity MCP 编译与 Console 检查。
3. Play 模式验证跨位置会合、120 秒超时、玩家抢占和完整轮播。
