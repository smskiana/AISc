# 协议公共层设计方案

> 执行记录: [execution.md](2026-07-14_协议公共层_execution.md)

## 需求理解

建立双端可扩展协议基础层，先以 `hello / hello_ack` 验证 envelope、版本协商、session、sequence 和错误结构，同时保持旧扁平消息兼容。

## Workstream / Roadmap

- Workstream: `ProtocolAndSave`
- Roadmap item: 协议与存档底座实施 / 协议公共层
- 相关 ADR: ADR-0006
- 工程口径变化: 新消息默认走 envelope；旧业务消息由兼容适配器继续接收

## 实现方案

1. Python 新增 codec、session、幂等缓存和协议错误模块。
2. WebSocket 入口先解码 envelope，再把规范化消息交给现有业务路由。
3. `GameRuntime` 只保留业务分派，握手交给独立 protocol handler。
4. Unity 新增 `ProtocolClient`，负责构造、解包、sequence 和握手。
5. Unity 通过官方 Newtonsoft JSON 包处理对象型 payload。
6. 旧扁平消息继续原样路由，不在本批迁移全部 DTO。

## 风险点

1. Unity 包安装会触发域重载，必须等待编译并检查 Console。
2. sequence 只在同一 session 内有意义，重连后从新 session 重新计数。
3. 幂等缓存先提供基础能力，本批不擅自套用到所有旧命令。

## 验证

1. Python 测试覆盖新旧解码、版本拒绝、sequence 和幂等缓存。
2. Unity 编译和 Console 无错误。
3. `hello / hello_ack` 使用 envelope 往返。
