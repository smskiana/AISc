# 已知问题独立索引方案

## Workstream

ProjectGovernance / Navigation

## Roadmap item

GOV / NAV

## 相关 ADR

1. `docs/DecisionRecords/ADR-0002-navigation-typed-path.md`

## 需求理解

当前 `docs/Workstreams/Navigation/README.md` 中已经记录了“传送稳定但入口触发偏宽”这个低优先级已知问题。用户要求“已知问题单开一个文档”，因此需要把已知问题从具体工作流 README 中抽出，建立统一入口。

目标：

1. 新增全局 `docs/KnownIssues.md`。
2. 将“传送稳定但入口触发偏宽”迁入该文档。
3. 导航工作流只保留链接，不继续承载问题细节。
4. 回写 `ProjectIndex.md` 和 `ChangeIndex.md`。

## 方案思路

### 1. 新增全局 KnownIssues

`docs/KnownIssues.md` 只记录暂不修或待验证的已知问题，不替代低级错误预防索引。

区别：

1. `KnownIssues.md`：当前仍存在、暂缓或待验证的问题。
2. `ErrorPreventionIndex.md`：已经修正、后续需要防复发的低级错误。

### 2. 工作流只保留入口

导航工作流的“已知问题”章节改为指向 `docs/KnownIssues.md`，避免同一问题在多个地方重复维护。

### 3. 回写索引

更新：

1. `docs/ProjectIndex.md`
2. `docs/AIChanges/ChangeIndex.md`

## 风险点

1. 不要把已修复的错误搬进 KnownIssues；已修复错误仍归 `ErrorPreventionIndex.md`。
2. KnownIssues 只做问题入口，不写成长篇调试日志。
3. 导航工作流不要重复保留同一问题的完整描述。

## 验证方式

1. 静态检查 `docs/KnownIssues.md` 存在。
2. 静态检查导航工作流指向 KnownIssues。
3. 静态检查 ChangeIndex 登记本轮执行记录。
4. 本轮不涉及代码编译。
