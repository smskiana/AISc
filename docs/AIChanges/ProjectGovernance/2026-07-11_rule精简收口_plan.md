# Rule 精简收口 — 设计方案

## 完成时间

2026-07-11

## 需求理解

用户认为当前 `AGENTS.md` / `CLAUDE.md` 中的规则已经积累了较多重复表述，虽然原则基本明确，但阅读成本偏高，容易让执行时抓不住重点。

本次目标不是改规则方向，而是：

1. 压缩重复表达
2. 保留原本已经确认的强制边界
3. 让各条规则更短、更好执行
4. 尽量不动会话启动、注释、plan / execution、低级错误这类结构性章节

## 本次修改范围

### 规则文件

- `AGENTS.md`
- `CLAUDE.md`

### 变更记录

- `docs/AIChanges/ProjectGovernance/2026-07-11_rule精简收口_plan.md`
- `docs/AIChanges/ProjectGovernance/2026-07-11_rule精简收口_execution.md`

## 具体方案

### A. 主要精简“核心行为底线”

重点压缩第 1 节的几条核心规则：

1. `最小修改`
2. `侵入控制`
3. `类减重与扩展性`
4. `Unity MCP优先`
5. `项目命名与索引规范`

精简方式：

1. 删除重复的解释性语句
2. 把并列条件改成更短的结构
3. 保留关键的“适用场景 / 非适用场景 / 违规判定”

### B. 保留规则优先级不变

本次不改变既有规则关系：

1. 修 bug / 小功能时，最小修改仍然强制
2. 新系统设计 / 结构治理时，类减重与扩展性优先
3. 资产层问题仍默认必须先走 Unity MCP

### C. 不扩展到其他文档

本轮不改：

- `docs/DesignDocs/ProjectNamingAndIndexing.md`
- `docs/ProjectIndex.md`
- 历史 execution 文档

只收口当前最常被直接读取的规则文件。

## 验证计划

1. `AGENTS.md` 与 `CLAUDE.md` 规则文本保持一致
2. 精简后仍保留：
   - 最小修改的适用边界
   - 类减重的强制触发
   - Unity MCP 的例外门槛
   - 命名规范入口
