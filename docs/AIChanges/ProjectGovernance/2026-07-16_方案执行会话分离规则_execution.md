# 方案执行会话分离规则执行记录

## Workstream

ProjectGovernance

## Roadmap item

GOV：项目治理

## 需求理解

用户要求复杂方案的制定与执行默认不得在同一对话中连续完成，避免长上下文下继续实现造成约束遗漏；当用户明确坚持同一对话执行时，仅允许在上下文较少且边界仍清晰的情况下例外继续。

## 实际改动

1. 更新 `AGENTS.md` 的“实际修改计划 & 执行文档”章节。
2. 新增“方案与执行会话分离（硬约束）”：
   - 复杂方案与执行默认分属两个独立对话。
   - 用户明确坚持时必须先做上下文负载评估。
   - 大量代码、日志、历史文档、多轮诊断、跨系统讨论或长方案均视为上下文较多，必须拒绝同对话执行。
   - 单文件小修、机械修改和无需复杂方案的任务不受限制。
