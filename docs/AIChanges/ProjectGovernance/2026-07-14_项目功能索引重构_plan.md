# 项目功能索引重构方案

> 执行记录: [execution.md](2026-07-14_项目功能索引重构_execution.md)

## 需求理解

当前项目索引混合了功能入口、时间顺序、最新交接和历史文件清单，`docs/AIChanges/` 根目录又集中堆放按日期命名的 plan / execution，导致 AI 容易先按时间翻阅，而不是先定位任务所属功能。

本轮目标是把项目文档入口改成“功能优先”：

1. 每个受维护目录用 `README.md` 简要说明目录功能和目录内容。
2. 项目总索引只按功能域提供入口，不再使用日期或最新 handoff 作为主导航。
3. `docs/AIChanges/` 的执行证据按功能目录存放，日期只保留为审计元数据。
4. 同步修改 AI 规则，使查找顺序稳定为“功能域 -> 目录说明 -> 当前口径 -> 必要历史证据 -> 代码调用链”。
5. 迁移后修复 Markdown 路径并执行链接检查。

## 所属 Workstream / Roadmap

- Workstream: `ProjectGovernance`
- Roadmap: `GOV` 项目治理
- 相关 ADR: ADR-0004；本轮新增 ADR-0005，补充功能目录优先的文档组织口径，不改变“暂不建设向量库”的决定。

## 实施范围

### 目录说明范围

为以下项目自维护目录维护入口说明：

- 项目根目录
- `docs/` 及其功能子目录
- `Assets/Scripts/` 及其直接功能子目录
- `backend/`、`backend/src/` 及其直接功能子目录
- `shared/`

Unity 生成目录、第三方包目录、缓存目录和纯素材叶子目录不机械添加 README，避免无效文档和 Unity `.meta` 噪音。

### AIChanges 功能目录

- `ProjectGovernance/`: 规则、索引、路线图、架构治理、错误预防
- `Memory/`: 记忆模型、图结构、检索、向量数据层、印象与遗忘
- `Navigation/`: 寻路、Anchor、传送、移动结果和位置语义
- `NpcBehavior/`: NPC 计划、行动选择、空闲表现和社交行为
- `Dialogue/`: 对话流程、快捷回复、称呼、现场感知和对话 UI
- `FrontendArchitecture/`: Unity 前端职责、生命周期、场景、Prefab、UI 与前端资产接入
- `ProtocolAndIntegration/`: WebSocket、前后端协议、命令和执行回报闭环
- `TestingAndDiagnostics/`: 跑测、诊断、性能、迁移验证和测试修复
- `Archive/Handoffs/`: 历史 handoff，仅用于追溯
- `artifacts/`: 原始日志、数据库和诊断产物，保持现状

跨多个功能域的记录只存放在一个主要功能目录，由其他功能索引引用，禁止复制文件。

## 方案步骤

1. 新增 ADR-0005，固定功能优先的目录和查找规则。
2. 建立 `docs/AIChanges/` 功能子目录及各自 `README.md`。
3. 根据主题和现有 ChangeIndex 分组迁移 plan / execution；handoff 迁入归档目录。
4. 重写 `docs/AIChanges/ChangeIndex.md`，使其只作为功能目录路由，不再按日期列长表。
5. 重写 `docs/ProjectIndex.md`，按功能域描述目录职责、内容和入口。
6. 为约定范围内缺少说明的目录补 `README.md`，已有 `Index.md` 的目录保留其详细文件索引，并由 README 指向。
7. 更新 `AGENTS.md`、命名索引规范、治理 Workstream、ADR 索引及相关旧入口。
8. 批量修复 Markdown 中指向已迁移 AIChanges 文件的路径。
9. 执行目录覆盖、plan / execution 互链和 Markdown 本地链接检查。

## 风险与处理

1. **历史链接失效**：建立旧路径到新路径的完整映射，批量替换后运行本地 Markdown 链接检查。
2. **跨系统记录归属不唯一**：选择主要变更目标作为物理归属，其他功能目录通过索引引用。
3. **目录说明过度扩张**：只覆盖项目自维护的功能目录和直接代码模块，不覆盖生成目录、第三方目录及纯素材叶子目录。
4. **日期重新成为导航主轴**：允许日期保留在文件名中用于审计，但 README、ChangeIndex 和 AI 查找规则不得按日期组织内容。
5. **历史文档陈述旧路径**：凡是可解析的真实 Markdown 路径全部更新；作为命令输出或历史描述出现的旧路径也尽量同步，避免后续 AI 误用。

## 验证标准

1. `docs/AIChanges/` 根目录不再堆放历史 plan / execution / handoff。
2. 每个约定范围内的功能目录存在 `README.md` 或明确的主索引入口。
3. `ProjectIndex`、`ChangeIndex` 和 `AGENTS.md` 均以功能域作为第一定位维度。
4. 全项目 Markdown 本地链接检查无因本轮迁移新增的失效链接。
5. 每对 plan / execution 的顶部互链能解析到真实文件。
6. `rg` 能从功能关键词稳定定位到对应 README、Workstream、ADR 和执行证据目录。
