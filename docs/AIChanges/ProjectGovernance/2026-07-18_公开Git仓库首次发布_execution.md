# 公开 Git 仓库首次发布执行记录

> 本次为小范围仓库治理修正，未单独创建 plan。

## 实际改动

1. 建立首个 Git 提交并发布到公开远端仓库。
2. 保留 Unity 客户端源码与必要资产、Python 后端、测试、共享配置和工程文档。
3. 在 `.gitignore` 中排除个人简历、后端日志、本地知识图状态、敏感配置、运行数据、商业 Sirenix 插件、未确认可再分发的字体资产，以及文件名嵌入本机绝对路径的异常副本。

## 安全边界

1. 两份简历包含真实姓名、手机号、邮箱和一串 API Key，整体不进入 Git 历史。
2. `backend/config/settings.yaml`、`backend/data/`、`backend/logs/`、`SaveData/` 与工具缓存不发布。
3. 发布前对最终暂存快照执行个人信息、常见密钥格式和 100 MB 超限文件复检。

## 文档与诊断评估

本次只改变版本控制和公开发布边界，不改变游戏运行时语义、协议、关联 ID 或 Unity 资产连线，因此 `aisc_debug` 与 `aisc_control` 不适用。

## 验证

1. 检查 Git 暂存文件清单与超限文件。
2. 扫描暂存内容中的个人姓名、联系方式、本机绝对路径和常见 API Key 格式。
3. 确认公开远端的默认分支和提交可读取。
