# 后端独立 30 分钟跑测 — 设计方案

## 需求理解

本轮目标是让后端在不启动 Unity / 前端的情况下可以独立跑测，并执行一次真实时间约 30 分钟的长测试。

重点不是联调 UI，而是验证：

1. 后端运行时不依赖前端 WebSocket 连接。
2. 没有 C# 接收时，NPC 可视化消息仍能进入后端日志。
3. 长测过程能稳定落盘 UTF-8 日志与 summary。
4. 跑测结束后可以看到性能与记忆相关关键日志。

## 方案思路

复用已有 `backend/scripts/run_7day_benchmark.py`，只做跑测稳定性补强：

1. 新增墙钟时间参数 `--wall-minutes`。
2. 当设置 `--wall-minutes` 时，脚本按真实时间运行，到点后优雅收尾并写 summary。
3. 默认禁用 HTTP 轮询队列，保证后端独立压测时不会积累无用前端消息。
4. 修正日志文件打开后又删除 artifact 目录的顺序问题。
5. summary 里补充完成的游戏分钟数、停止原因、墙钟限制，便于复盘。

## 风险点

1. 如果午夜 LLM 阶段正在执行，墙钟到点后需要等当前阶段结束才能优雅写 summary。
2. 如果外部 LLM / embedding 服务不可用，跑测可能暴露为错误日志；这属于本次长测需要观察的内容。
3. 30 分钟真实时间跑测期间，终端输出可能仍受宿主控制台编码影响，但落盘日志应保持 UTF-8。
