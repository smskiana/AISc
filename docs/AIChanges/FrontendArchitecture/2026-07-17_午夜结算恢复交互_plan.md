# 午夜结算恢复交互：执行方案

## 需求与边界

- 午夜结算后端成功、自动存档成功且暂停解除后，玩家仍不可移动。
- 仅修复 Unity 的 `gameplay_ready` 成功终态，不改场景、Prefab、协议字段或后端结算逻辑。

## 当前口径

- Workstream：`docs/Workstreams/FrontendArchitecture/README.md`、`docs/Workstreams/ProtocolAndSave/README.md`。
- `MIDNIGHT_SETTLEMENT_COMPLETE` 表示目标游戏日 NPC 日计划已准备完成，可解除 Gameplay 锁。

## 实施步骤

1. 为 `GameStateStore` 增加显式的 ready 状态写入入口。
2. 由 `GameManager` 在收到午夜成功终态时恢复 ready，再转发既有睡眠流程。
3. 增加 EditMode 回归测试，验证 ready 状态可从结算锁恢复。
4. 用 Unity MCP 编译与 EditMode 测试验证。

## 风险与验收

- 风险：过早解除锁会允许在次日准备前操作；因此只绑定后端成功终态。
- 验收：午夜进度锁定后，成功终态使 `IsGameplayReady` 为 true；Unity 无编译错误，相关 EditMode 测试通过。
