# 前端职责拆分类补齐方案

## Workstream

FrontendArchitecture

## Roadmap item

FEA

## 需求理解

按用户指定顺序，在已完成 `GameCommandSender` 后继续补齐当前代码中缺失的三个前端职责类：`GameStateStore`、`NpcBehaviorApplier`、`NpcSocialRendezvousController`。本轮只读当前代码与必要索引，不翻旧记录，不改后端协议，不做场景或 Prefab 连线。

## 方案思路

1. `GameStateStore`：从 `GameManager` 拆出世界状态字段写入与查询，`GameManager` 保留公开属性、事件和外部 API。
2. `NpcBehaviorApplier`：从 `NpcSpawner` 拆出 `NPC_BEHAVIOR` 如何落到 `NpcEntity` 的行为应用逻辑。
3. `NpcSocialRendezvousController`：从 `NpcBubbleManager` 拆出 NPC-NPC 社交靠近、等待距离与移动锁控制。
4. 保持现有 MonoBehaviour 挂载结构不变，新增类优先做普通 C# 协作者，由现有组件在运行时 new 出来。

## 风险点

1. `GameManager` 的公开属性和事件已有多个调用点，外部签名必须保持。
2. `NpcBubbleManager` 的气泡队列与协程不能被拆断，否则会影响 NPC 对话轮播。
3. 新脚本需要通过 Unity AssetDatabase 导入生成 `.meta`，否则 Unity 编译域会找不到类型。

## 验证方式

1. `rg` 检查旧类是否仍承担被拆出的直接职责。
2. `dotnet build AISc.sln --no-restore`。
3. Unity MCP 导入新脚本、刷新编译、`validate_script` 校验相关脚本，并读取 Console 错误。
