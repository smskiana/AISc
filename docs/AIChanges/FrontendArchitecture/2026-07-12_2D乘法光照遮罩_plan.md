# 2D 乘法光照遮罩 — 方案

## 需求理解

本轮需要新增一个可用于 2D 画面的遮罩 shader，使遮罩颜色与下方已经渲染的元素做乘法混合，用来模拟日夜、阴影、室内暖光等光照变化。

## 方案思路

1. 项目当前使用 Built-in 渲染管线，因此采用传统 ShaderLab / CG 写法。
2. 使用透明队列与 `Blend DstColor Zero`，让遮罩输出色直接乘到底色。
3. 遮罩贴图 alpha 和材质 `Strength` 控制影响范围；alpha 为 0 或强度为 0 时输出白色，保持底图不变。
4. 保留 `Stencil` / `ColorMask` / UI clip 相关属性，使材质既可用于 SpriteRenderer，也可用于 UGUI Image 覆盖层。
5. 附带一个默认材质，方便后续直接给全屏 Sprite 或 Canvas Image 试用。

## 涉及文件

- `Assets/Shaders/Shader_2DMultiplyLightingMask.shader`
- `Assets/Materials/M_2DMultiplyLightingMask.mat`
- 对应 Unity `.meta` 文件

## 风险点

1. 乘法混合主要用于压暗和染色，不能把底图无损提亮到超过原始亮度。
2. 需要保证该遮罩对象绘制在被影响元素之后；SpriteRenderer 可通过 Sorting Layer / Order 控制，UI 可放在目标层级上方。
3. Unity 未在本轮启动导入，shader 编译需要后续在编辑器中确认 Console 无报错。

## 验证方式

1. 静态检查 shader 文件存在、语法结构完整。
2. 确认默认材质引用新增 shader。
3. 后续在 Unity 中将材质挂到全屏 Sprite 或 UI Image，观察白色区域不变、灰/彩色区域乘法压暗或染色。
