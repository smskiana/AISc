# MSYH 字体统一 — 设计方案

## 需求理解

当前 Unity 中部分 TextMeshPro 文本仍不支持中文显示，用户明确要求统一改用：

- `Assets/Fonts/MSYH SDF.asset`

从当前资产状态看：

1. 开始界面预制体已在使用 `MSYH SDF`
2. 对话 UI 预制体 `UI_DialogueCanvas.prefab`
3. 选项按钮 `UI_ChoiceButton.prefab`
4. NPC 名字牌相关预制体
5. `Town_Main.scene` 中一批旧 TMP 文本

仍然挂着旧字体资源。

## 方案原则

1. 优先资产层修复，不在运行时脚本里动态补字体
2. 未来 Builder 生成的新 UI 也要显式挂 `MSYH SDF`
3. 已存在的场景 / Prefab 通过编辑器工具一次性归一化

## 具体方案

### 1. Builder 显式字体化

在：

1. `DialogueUIBuilder`
2. `StartMenuBuilder`

中统一：

1. 通过 `AssetDatabase.LoadAssetAtPath<TMP_FontAsset>("Assets/Fonts/MSYH SDF.asset")`
2. 给新建的 `TextMeshProUGUI` / `TMP_InputField` 相关文本节点显式设置：
   - `font`
   - `fontSharedMaterial`

避免继续依赖 TMP 默认字体。

### 2. 现有资产归一化

新增 `TmpChineseFontNormalizer` 菜单工具，例如：

- `Tools/Apply MSYH SDF Font`

职责：

1. 扫描当前打开场景中的 `TMP_Text`
2. 扫描：
   - `Assets/Prefabs/UI`
   - `Assets/Prefabs/NPCs`
3. 把所有找到的 `TMP_Text` 统一切到 `MSYH SDF`
4. 一并同步字体材质

### 3. MCP 执行

脚本完成后通过 Unity MCP：

1. 刷新编译
2. 执行 `Tools/Apply MSYH SDF Font`
3. 如有需要再执行：
   - `Tools/Build Dialogue UI`
   - `Tools/Build Start Menu`
4. 检查 Console 和实际场景 / Prefab 引用

## 风险点

1. 旧 Prefab 中个别文本可能依赖特殊材质效果，统一材质时要注意不要误伤
2. 如果有非中文专用英文字体样式需求，本轮会统一为中文兼容优先

## 验证计划

1. 新增脚本编译通过
2. `UI_DialogueCanvas.prefab`、`UI_ChoiceButton.prefab`、NPC 预制体字体引用切到 `MSYH SDF`
3. `Town_Main.scene` 中遗留 TMP 文本字体也切到 `MSYH SDF`
4. Console 无新增 error
