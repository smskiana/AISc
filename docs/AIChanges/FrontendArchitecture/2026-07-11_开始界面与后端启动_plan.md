# 开始界面与后端启动 — 设计方案

## 需求理解

当前 Unity 场景一启动就直接尝试连接 WebSocket 并自动发 `GAME_START`，没有真正的开始界面，也没有“登录 / 读取存档 / 启动后端”的用户入口。

本次目标是在不手工重搭场景的前提下，补一个可运行的开始界面，支持：

1. 本地登录名输入
2. 自动检查 / 启动 Python 后端
3. 连接后端后读取存档列表
4. 选择新游戏或加载指定存档
5. 在真正进入 `GAME_READY` 前锁住玩家输入

## 已确认现状

- Unity 当前只有一个场景：`Assets/Scenes/Town_Main.scene`
- `WebSocketClient` 目前 `Start()` 就自动健康检查并尝试连接
- `GameManager.Start()` 当前会等待 WS 成功后自动发 `GAME_START`
- 后端 `runtime.py` 已支持：
  - `GAME_START`
  - `GET_SAVES`
  - `LOAD_REQUEST`
  - `SAVE_REQUEST`
- 后端 `SaveManager.list_saves()` 已能返回可直接渲染的存档列表

## 方案原则

1. 不直接手改现有场景 YAML，优先运行时生成开始界面
2. 把“连接后端 / 请求存档 / 进游戏”收口为一条清晰前端状态流
3. 登录先做本地玩家名入口，不扩成真正账号系统
4. 尽量兼容现有 GameManager / WebSocketClient 主链，不重写通信层

## 具体方案

### 1. 让 WS 与 GAME_START 改成“手动驱动”

当前最大问题是：

- `WebSocketClient` 自动连
- `GameManager` 自动开局

这会让开始界面来不及接管流程。

因此改为：

- `WebSocketClient` 支持 `BeginConnect()`
- 自动连接改成可配置，默认关闭
- `GameManager` 不再默认自动发 `GAME_START`
- 新增前端方法：
  - `RequestSavesList()`
  - `StartNewGameFlow()`
  - `StartLoadFlow(slot)`

### 2. 新增运行时开始界面控制器

新增 `StartMenuController`，通过 `RuntimeInitializeOnLoadMethod` 在场景加载后自动创建。

它负责：

- 运行时生成一个 Overlay Canvas
- 构建标题、状态文案、登录输入框、按钮区、存档列表区
- 绑定按钮行为：
  - 登录
  - 启动 / 重试后端
  - 新游戏
  - 刷新存档
  - 点击某条存档进行加载

### 3. 同步启动后端

开始界面启动时：

1. 先访问 `/api/health`
2. 若后端已在线，则直接连 WS
3. 若不在线，则尝试启动 `backend/run.py`
4. 健康检查通过后再调用 `WebSocketClient.BeginConnect()`

实现上：

- 从 Unity 工程根目录相对定位 `backend/run.py`
- 尝试使用本机 `python` 或 `py -3` 启动
- 启动成功后轮询 health，直到可连接或超时

### 4. 存档加载流程

连接成功后开始界面主动发送：

- `GET_SAVES`

拿到 `SAVES_LIST` 后渲染存档按钮。

点击加载时流程为：

1. 发送 `LOAD_REQUEST`
2. 收到 `LOAD_COMPLETE`
3. 再发送 `GAME_START(CONTINUE)`
4. 等 `GAME_READY` 后关闭开始界面

这样可以兼容当前后端“load 只恢复数据库，game_start 才重新推完整世界状态”的实现口径。

### 5. 进入游戏前锁输入

新增 `GameManager.IsGameplayReady`。

只有在收到 `GAME_READY` 后才设为 `true`。

`PlayerController.Update()` 在 `IsGameplayReady == false` 时不处理移动和点击，避免开始界面阶段玩家在背景里乱动。

## 风险点

1. 本地 Python 命令若不在 PATH，后端自动启动可能失败，需要状态文案明确提示
2. 开始界面当前是运行时生成 UI，不走现有 prefab / scene 资源配置流
3. “登录”本轮只做本地玩家名，不会立即联动到后端对白里对玩家的称呼

## 验证计划

1. Unity 脚本编译通过，无新增错误
2. 启动游戏后先显示开始界面，而不是直接进城镇
3. 若后端未启动，开始界面可自动或手动拉起后端
4. 连接成功后能刷新并显示存档列表
5. 点击新游戏或加载存档后，收到 `GAME_READY` 再进入正式游戏
