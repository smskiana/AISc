# 玩家基础 UI 与日常闭环实施计划

> 执行记录: [execution.md](2026-07-16_玩家基础UI与日常闭环_execution.md)

## 需求理解

按 `docs/Workstreams/FrontendArchitecture/PlayerBasicUIExecutionPlan.md` 分阶段建立玩家基础 UI、Unity 权威时间、暂停、系统消息、背包、存档管理、退出、睡眠跨日和致命错误处理的完整日常闭环。

用户已明确授权对 `GameManager` 与 `GameStateStore` 进行职责减重。实现不得把新增 UI、时间、暂停、存档产品逻辑或睡眠流程继续堆入这两个类。

## 所属工作流

1. 主要功能域：`FrontendArchitecture`。
2. 关联功能域：`ProtocolAndSave`，仅承接时间镜像、暂停同步、午夜结算和存档产品层所需协议。
3. 执行依据：`docs/Workstreams/FrontendArchitecture/PlayerBasicUIExecutionPlan.md`。

## 相关决策

1. `ADR-0003-unity-mcp-asset-first.md`：场景、Prefab、UI、字体和引用连线通过 Unity MCP 完成。
2. `ADR-0006-unity-authoritative-save-memory-checkpoint.md`：Unity 保持世界存档权威，Python 只保存 AI 记忆检查点。

## 实施顺序

1. 固化现有编译、测试、场景、菜单、对话、传送和存档基线。
2. 建立多来源暂停 Model / Controller 与 Python 即时同步接口。
3. 建立 Unity 权威时间 Model / Controller / View、Python 镜像同步和昼夜 View。
4. 建立普通提醒、阻塞弹窗和加载遮罩模块。
5. 建立地点 HUD 与只读背包 MVC，并接入现有存档 DTO。
6. 在现有存档底座上建立自动槽和无限手动槽的产品管理层。
7. 建立暂停菜单、设置占位页、返回菜单和退出桌面流程。
8. 建立独立睡眠 Controller、午夜结算、床位与醒来 Anchor、自动存档。
9. 建立后端断线和午夜结算失败/超时的致命错误闭环，完成完整 Play 回归。

## 预计涉及文件

1. Unity 运行时代码：`Assets/Scripts/Core/`、`Assets/Scripts/UI/`、`Assets/Scripts/Save/`，并按职责新增独立模块目录。
2. Unity 资产：开始菜单、`Town_Main`、喫茶店场景、UI Prefab、Canvas、床位触发区、Anchor、昼夜材质或 Shader 配置。
3. Python：应用路由、时间镜像、暂停与午夜结算相关模块及测试。
4. 文档：前端与协议存档 Workstream、相关代码目录 README、本执行记录。

## 风险与控制

1. 方案跨度较大，严格按阶段验收，未通过的阶段不标记完成。
2. 现有存档格式和重连协议已经稳定，新增产品层不得破坏既有事务和权威边界。
3. 所有静态 UI 和场景连线通过 Unity MCP 完成，不使用运行时动态生成结构兜底。
4. NPC 对话、任务和社交已有暂停与真实单调时间语义，迁移游戏时间时需避免影响协议超时。
5. 新增函数与实质修改函数补功能注释，保留全部既有注释。
6. 每阶段完成后检查功能文档与代码现状是否一致。
7. 开发期临时策略：每次确认进入新游戏前，先永久删除全部 Unity 主存档与 Python 记忆检查点；任一端清理失败则不得进入新游戏。该策略后续恢复正式多存档流程时必须显式移除。

## 验证方式

1. Unity Console 编译检查与相关 EditMode / PlayMode 测试。
2. Python 定向测试与协议测试。
3. Unity MCP Play 模式逐阶段验收。
4. 最终按专项方案第 13 节完整路径进行人工可观察回归。
