# 玩家物理移动接入 - 设计方案

## 需求理解

当前玩家 WASD 移动直接改 `transform.position`，不会经过 Unity 2D 物理求解，因此场景碰撞体无法阻挡玩家。

本轮目标是让玩家移动接入 Unity 2D 碰撞系统，保持现有输入、对话距离检测和位置同步行为不变。

## 相关入口

- Workstream: `docs/Workstreams/Navigation/README.md`
- 相关代码: `Assets/Scripts/Core/PlayerController.cs`
- 相关资产: `player_dy` 场景实例、`Assets/Prefabs/Player/Player_Main.prefab`
- 相关规范: `docs/DesignDocs/UnityMCPUsageRules.md`

## 方案

1. `PlayerController` 保留 `Update()` 做输入采样、点击 NPC 和位置同步。
2. 新增 `Rigidbody2D` 引用，在 `FixedUpdate()` 中使用 `Rigidbody2D.MovePosition()` 执行位移。
3. 玩家移动时将刚体设置为 `Dynamic`、关闭重力、冻结旋转，避免 2D 顶视角玩家受重力或转动影响。
4. 通过 Unity MCP 给当前场景玩家实例补 `Rigidbody2D` 并设置物理参数。
5. 同步补齐 Player Prefab 的 Rigidbody2D/Collider2D 配置，避免后续实例化回退到非物理玩家。

## 风险点

1. 如果场景障碍物 Collider2D 被设置为 Trigger 或 layer collision matrix 排除了 player 层，仍可能需要后续资产层排查。
2. `MovePosition()` 会参与物理求解，但高速移动时仍可能需要调小速度或启用连续碰撞检测；本轮先保持移动速度不变。
3. 当前玩家位置同步仍按 Anchor 半径节流，不改变后端 location 上报语义。
