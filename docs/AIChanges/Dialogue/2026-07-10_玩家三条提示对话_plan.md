# 玩家三条提示对话 — 设计方案

## 需求理解

目标是让玩家和 NPC 对话时：

1. 每轮 NPC 说完后，前端能看到 3 条可点击的建议回复
2. 玩家仍然保留自由输入任意文本的能力
3. 尽量复用现有对话协议和 UI，不重做一套新系统

## 已确认现状

### 前端

`PortraitDialogueUI` 已经支持：

1. 显示 `choices`
2. 同时显示自由输入框
3. 点击选项后走同一条 `PLAYER_CHOICE` 通道

所以前端结构本身已经满足“选项 + 自由输入并存”。

### 后端

`PlayerDialogueService` 当前会在 `DIALOGUE_COMPLETE` 中返回：

1. `npc_id`
2. `npc_text_complete`

但没有稳定填充 `choices`，导致前端虽然有选项容器，实际大多数时候不会出现 3 条建议回复。

## 方案原则

1. **最小改动**：不扩 WebSocket 协议，不新增新的消息类型
2. **职责清晰**：
   - Python 负责根据对话上下文生成建议回复
   - Unity 继续只负责展示与交互
3. **失败可降级**：即使建议生成失败，也不能影响正常自由输入对话

## 涉及文件

1. `backend/src/application/dialogue_service.py`
2. `backend/src/dialogue/prompt_builder.py`
3. `Assets/Scripts/Dialogue/PortraitDialogueUI.cs`
4. `docs/AIChanges/Dialogue/2026-07-10_玩家三条提示对话_plan.md`
5. `docs/AIChanges/Dialogue/2026-07-10_玩家三条提示对话_execution.md`

## 具体方案

### 1. 后端在每轮 NPC 回复后生成 3 条建议回复

在 `PlayerDialogueService` 中新增一个统一的建议生成流程：

1. 对话首句完成后生成 3 条建议
2. 玩家回复后，下一轮 NPC 回复完成时再生成 3 条建议
3. 把建议列表写入 `DIALOGUE_COMPLETE.choices`

这样前端现有 `OnNpcReplyComplete(msg.choices)` 就能直接工作。

### 2. PromptBuilder 负责组装“建议回复”提示词

在 `PromptBuilder` 中新增一个轻量提示词构造方法，输入：

1. NPC ID
2. target ID
3. 当前 NPC 完整回复
4. 已有对话上下文

输出一个专门用于“给玩家生成 3 条建议回复”的 messages 列表。

生成要求：

1. 每条建议 6-24 字左右
2. 要自然、可直接点选
3. 三条风格略有区分，例如：
   - 顺着聊
   - 追问/关心
   - 轻松/转话题
4. 不能替玩家锁死，只作为提示

### 3. 生成失败时回退到规则化兜底

如果 LLM 生成失败或解析失败：

1. 仍返回 3 条兜底建议
2. 不阻断 `DIALOGUE_COMPLETE`
3. 玩家始终还能自由输入

兜底方向：

1. 顺着回应
2. 问一句细节
3. 表达情绪或转到轻松话题

### 4. 前端只做小补强

`PortraitDialogueUI` 这轮不重做结构，只补一个小体验：

1. 回复提示文案明确说明“可点选建议，也可自由输入”

这样玩家能立刻理解这两个入口同时可用。

## 风险点

1. LLM 建议回复可能偶尔太长或不够像玩家口吻，需要在解析后做裁剪
2. 若建议生成和 NPC 回复完全串行，会增加单轮对话完成到开放输入的耗时
3. `JsonUtility` 侧不需要改结构，但后端必须保证 `choices` 始终是字符串数组或省略为空

## 验证计划

1. 后端 `DIALOGUE_COMPLETE` 能稳定带上 3 条 `choices`
2. Unity 前端在 NPC 每轮说完后显示 3 个可点击选项
3. 点击选项后还能正常继续对话
4. 不点击选项，直接在输入框自由输入也能继续对话
5. 建议生成失败时，对话仍可通过自由输入继续
