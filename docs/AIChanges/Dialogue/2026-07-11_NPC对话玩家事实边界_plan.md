# NPC 对话玩家事实边界 — 设计方案

## 问题

30 分钟后端独立跑测暴露了比性能/计划过滤更严重的问题：NPC 会把没有发生过的玩家行为说成事实。

典型日志：

```text
千早: 对了对了，我刚才路过车站前，看到小李在搬招牌了，是不是真的要重开啦？
```

这会造成玩家体验割裂，因为玩家并没有执行“搬招牌”这个行为。

## 根因判断

当前长期记忆里有合法事实：

1. 小李小时候在樱桥通长大。
2. 小李后来去了城市，现在回来了。
3. 小李准备重新开奶奶留下的喫茶店。
4. 奶奶去世后喫茶店关了很久。

但 NPC 对话 prompt 没有明确区分：

1. 已知背景事实
2. NPC 推测
3. NPC 亲眼所见
4. 玩家实际已发生行为

LLM 因此会把“准备重开喫茶店”扩写成“看到玩家搬招牌/收拾店/帮玩家搬东西”等未发生事件。

## 修复方案

采用双层防线：

### 1. Prompt 层事实边界

在玩家对话和 NPC ↔ NPC 对话 prompt 中加入“玩家事实边界”：

1. 可说的玩家事实白名单。
2. 禁止把玩家未发生动作说成亲眼所见。
3. 若要谈进展，只能说“不清楚具体进展 / 听说准备重开”。
4. 禁止生成“看到小李搬招牌/帮小李搬东西/小李正在收拾”等内容。

### 2. 输出层守门

在 NPC ↔ NPC 对话解析后、发送气泡前增加轻量事实守门：

1. 检测“小李/玩家 + 搬/收拾/修/贴/挂/打扫/整理/帮”等未证实动作。
2. 命中后将台词替换成保守表达。
3. 写入日志 `[FACT_GUARD]`，便于跑测统计。
4. 被替换后的台词再进入气泡发送、短期记忆和状态影响，避免污染记忆库。
