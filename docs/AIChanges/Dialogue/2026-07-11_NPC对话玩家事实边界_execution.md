## 关键实现

### 1. Prompt 层新增玩家事实边界

现在 prompt 明确告诉 LLM：

1. NPC 只能确定“小李小时候在樱桥通长大 / 从城市回来 / 准备重开奶奶的喫茶店 / 喫茶店曾经关闭”。
2. 不允许说“看到小李搬招牌”“小李正在收拾店”“刚帮小李搬东西”等未发生行为。
3. 如果要聊喫茶店进展，只能说“听说准备重开 / 具体进展还不清楚”。

### 2. 输出层新增 `[FACT_GUARD]`

NPC ↔ NPC 对话在解析后、发送气泡前会检查：

1. `看到/看见/路过/碰到/遇见 + 小李/玩家 + 搬/收拾/整理/打扫/修/贴/挂/安装/装修/招牌`
2. `小李/玩家 + 搬/收拾/整理/打扫/修/贴/挂/安装/装修/招牌`
3. `帮/陪 + 小李/玩家 + 搬/收拾/整理/打扫/修/贴/挂/安装/装修`
4. `喫茶店/咖啡店/店里/那家店 + 搬/收拾/整理/打扫/修/贴/挂/安装/装修/招牌`

命中后替换为：

```text
听说小李准备重新开奶奶留下的喫茶店，不过具体进展我还没确认。
```

并写日志：

```text
[FACT_GUARD] npc_dialogue_player_claim sanitized ...
```

被替换后的文本才会进入气泡发送、短期记忆和即时印象，避免污染后续记忆。

## 验证

### 1. Python 编译

执行：

```powershell
python -m py_compile backend/src/dialogue/prompt_builder.py backend/src/npc/npc_dialogue.py
```

结果：通过。

### 2. 守门 smoke

输入坏台词：

```text
我刚才路过车站前，看到小李在搬招牌了，是不是真的要重开啦？
诶嘿嘿~今天去帮小李搬了点东西，路过这儿看见你在就想打个招呼！
听说那家喫茶店开始挂招牌了。
```

结果：全部被替换为保守表达。

输入合法背景：

```text
听说小李准备重新开奶奶留下的喫茶店。
```

结果：未被误伤。

## 结论

这次修复把“玩家未发生行为”从单纯 prompt 约束提升为 prompt + 输出守门双层机制。

后续跑测重点看：

1. `[FACT_GUARD]` 是否还有触发。
2. NPC_BUBBLE 中是否仍出现玩家未执行过的动作。
3. 如果玩家后续真的执行了“搬招牌 / 收拾店”等任务，再把玩家行为事件写入短期记忆/图记忆，让守门逻辑可以根据真实记忆放行。
