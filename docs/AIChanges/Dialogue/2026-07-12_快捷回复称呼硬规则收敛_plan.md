# 快捷回复称呼硬规则收敛 — 设计方案

## 需求理解

当前快捷回复为了修“和叶姐 / 她”问题，额外加入了很硬的性别称呼禁令。现在完整现场感知层已经接入，继续保留这些禁令会让 prompt 显得死板，并可能影响自然表达。

本轮目标是去掉硬性禁止句，把称呼约束收敛为柔和的称呼建议。

## 方案思路

1. `PLAYER_REPLY_SUGGESTION_SYSTEM` 去掉“不要凭名字猜性别 / 不要叫姐”这类硬规则。
2. 保留“参考现场感知和人物信息，称呼自然”。
3. NPC / 玩家 profile 的 `address_hint` 从禁令改成正向称呼建议。
4. 保留 `visual_tags`、`presence_tags`、`speech_perception`，让感知层自然提供性别与存在感。

## 涉及文件

- `backend/src/dialogue/prompt_builder.py`
- `backend/config/npc_profiles/*.json`
- `backend/config/player_profile.json`

## 验证

1. JSON 校验
2. Python 编译
3. Prompt smoke 确认不再出现硬性“不要叫和叶姐”文本，同时仍保留和叶男性视觉感知。
