# NPC对话气泡轮播 — 设计方案

## 需求理解

当前 AI 驱动的 NPC 对话中，头顶气泡有时会同时挂在两个人身上，视觉上像是“重叠播报”，不符合“你一句我一句”的轮流说话预期。

本次目标是把 NPC 对话气泡改成按对话参与者轮流显示：

1. 同一组 NPC 的对话只允许同时存在一个气泡
2. 新一句出现前，上一句对应的气泡应先消失
3. 仍保留不同无关对话组并行显示的能力，避免把全场所有气泡都串成一条全局队列

## 已确认现状

- `Assets/Scripts/Dialogue/NpcBubbleManager.cs` 当前只按 `npc_id` 做单人队列
- 也就是说：同一个 NPC 自己不会同时播两条，但 `A → B` 和 `B → A` 会分别在两人头上同时显示
- `backend/src/npc/npc_dialogue.py` 已按轮次顺序发送 `NPC_BUBBLE`，但句间延迟只有 `0.5s`，远小于气泡显示时长，因此前端若不做轮播就会叠住
- `BubbleUI.Show()` 当前是单个 NPC 自己的显示/隐藏逻辑，不知道“对话双方”的上下文

## 方案原则

1. 最小修改，优先修前端显示逻辑，不改后端协议
2. 轮播粒度按“对话参与者组合”划分，而不是按全局或单个 NPC
3. 气泡显隐规则集中在 `NpcBubbleManager`，不把配对逻辑散到 `BubbleUI`

## 具体方案

### 1. 把队列从“按 npc_id”改成“按对话 key”

为每条 `NPC_BUBBLE` 构造稳定对话 key：

- 双人对话：按 `npc_id` 与 `target_npc_id` 排序后拼接
- 单人气泡：退化为自己的 key

这样 `A -> B` 和 `B -> A` 会进入同一条会话队列，天然实现轮流显示。

### 2. 显示新一句前先清理双方旧气泡

在取出下一条气泡准备显示时：

- 隐藏说话方当前气泡
- 隐藏目标方当前气泡

然后只显示当前说话方的新气泡。

### 3. 在气泡管理器里串行推进会话

每个对话 key 维护：

- 一条消息队列
- 一个“当前是否播放中”的标记

显示完当前句子并等待一小段缓冲后，再推进下一句。

### 4. 玩家打开正式对话时清掉该 NPC 头顶气泡

当玩家与某 NPC 打开 `PortraitDialogueUI` 时，如果他头顶还残留世界空间气泡，会影响阅读。

因此在 `GameManager.StartDialogue()` 中增加一个轻量事件，让 `NpcBubbleManager` 在玩家开聊时清掉该 NPC 相关的世界气泡。

## 风险点

1. 如果后端未来发送三人或更复杂的多人对话，当前 key 仍按双人对话假设处理
2. 如果气泡时长估算偏长，前端轮播会更稳，但节奏可能显得略慢

## 验证计划

1. Unity 脚本编译通过，无新增错误
2. 同一组 NPC 对话时，只出现一个头顶气泡
3. A 说完后，A 的气泡消失，再轮到 B 显示
4. 玩家主动与某 NPC 开聊时，该 NPC 旧的世界气泡会被清掉
