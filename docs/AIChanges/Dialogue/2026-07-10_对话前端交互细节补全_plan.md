# 对话前端交互细节补全 — 设计方案

## 需求理解

当前玩家与 NPC 的对话前端存在三个直接影响可玩性的问题：

1. NPC 说完一句后，前端没有可输入的玩家回复框，导致对话无法继续
2. 对话进行时，玩家仍可继续移动 / 再次点击交互，缺少“对话中锁定”
3. 下一轮 NPC 回复开始时，没有稳定清空上一轮台词，导致文本串在一起

本次目标是在不改后端协议的前提下，把 Unity 前端对话体验补成一个可持续进行的完整闭环。

## 已确认现状

### 后端协议

后端 `PlayerDialogueService` 已支持：

- `DIALOGUE_START`
- `PLAYER_CHOICE`
- `DIALOGUE_COMPLETE`
- `DIALOGUE_END`

其中 `PLAYER_CHOICE` 的 `choice_text` 本质就是玩家自由输入文本，不依赖固定选项，因此前端可以直接增加输入框，无需扩协议。

### Unity 前端

- `PortraitDialogueUI` 当前只有 NPC 文本展示和可选 `choices`
- `GameManager` 仅在 `msg.choices` 非空时调用 `ShowChoices()`
- `PlayerController` 当前不会因为对话开启而锁移动
- 当前“清空台词”逻辑只发生在点选选项时，不足以覆盖自由输入场景

## 修复原则

1. **最小侵入**：不改后端消息结构，不重做整套 UI
2. **当前场景可直接生效**：优先让现有 `DialogueCanvas` 不用手工重搭也能工作
3. **保留兼容性**：如果后端未来重新提供 `choices`，前端仍能显示
4. **职责清晰**：
   - `GameManager` 管对话状态与协议发送
   - `PortraitDialogueUI` 管展示、输入和交互按钮
   - `PlayerController` 只根据全局对话状态决定是否允许移动

## 具体方案

### A. 给对话 UI 补自由输入区

在 `PortraitDialogueUI` 内增加运行时可自建的回复区：

- 输入框 `TMP_InputField`
- `发送` 按钮
- `结束对话` 按钮

实现方式：

- 若场景中已配置引用，则优先使用
- 若当前场景还没有这些对象，则在运行时自动创建最小可用 UI

这样可以避免为了这次修复强依赖手工改场景或重跑 UI Builder。

### B. 让 NPC 每轮说完后开放回复

调整 `GameManager` 的 `OnDialogueComplete` 回调：

- 不再只在 `choices` 非空时处理
- 改为统一通知 `PortraitDialogueUI` “当前这轮 NPC 已说完”

`PortraitDialogueUI` 在这个时机：

- 停止打字机状态
- 展示输入区
- 若有 `choices` 也一并展示

### C. 在玩家发送下一句前清空旧 NPC 台词

把“清空上一轮台词”的逻辑从“点选选项时顺手处理”提升为正式流程：

- 玩家点击 `发送`
- 或玩家点击某个 `choice`

都会先执行：

- 清空 `_dialogueText`
- 清空 `_accumulatedText`
- 重置打字机索引
- 隐藏回复控件

然后再发 `PLAYER_CHOICE`，确保下一轮 NPC 文本从空白开始。

### D. 加入对话中锁移动

在 `GameManager` 增加 `IsDialogueActive` 状态：

- `StartDialogue()` 成功打开 UI 时设为 `true`
- 本地请求结束对话或收到 `DIALOGUE_CLOSE` 时设为 `false`

`PlayerController.Update()` 根据该状态：

- 对话中不处理 `WASD` 移动
- 不处理再次点击 NPC

### E. 提供本地结束出口

为避免“锁住后无法退出”，在 UI 中提供：

- `结束对话` 按钮
- `Esc` 快捷结束

触发时调用 `GameManager` 统一结束接口，本地先解锁 UI / 移动，再通知后端结束对话。

## 风险点

1. 运行时创建 `TMP_InputField` 需要确保场景里有可用 `Canvas` / `EventSystem`
2. 现有 `DialogueCanvas` 默认 inactive，运行时补 UI 时要兼容隐藏状态
3. 若后端异常路径只发 `DIALOGUE_TOKEN(is_complete=true)` 而不发 `DIALOGUE_COMPLETE`，输入开放时机仍可能受限，但不影响主路径修复

## 验证计划

1. Unity 编译通过，无新增脚本错误
2. 点击 NPC 后：
   - 玩家停止移动
   - 对话 UI 正常打开
3. NPC 首句结束后：
   - 输入框出现
   - 可输入文本并发送
4. 发送后下一轮 NPC 回复：
   - 从空白文本开始，不和上一轮串联
5. 点击结束对话或按 `Esc`：
   - UI 关闭
   - 玩家恢复移动
