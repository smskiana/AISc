# 玩家昵称语义收敛 — 设计方案

## 需求理解

当前“小李”被作为玩家固定显示名硬塞进 prompt，读起来像系统指定本名。实际设计上它更适合作为玩家默认昵称；正式语义应是“玩家 / 归乡的喫茶店继承者”，昵称可以被 NPC 自然使用。

## 方案思路

1. `player_profile.json` 增加 `nickname`，将 `name` 改为“玩家”。
2. 现场感知中玩家显示为“玩家（昵称：小李）”，而不是“小李：未知”。
3. `PromptBuilder` 的玩家显示名从固定“小李”改为读取昵称；涉及叙述时使用“玩家”避免僵硬。
4. 新增统一昵称渲染工具，在 prompt/messages 返回前替换 `{player_nickname}`、`{player_label}` 与旧默认昵称。
5. 不大规模改历史设计文档和旧跑测日志，只处理运行时代码与当前配置。
