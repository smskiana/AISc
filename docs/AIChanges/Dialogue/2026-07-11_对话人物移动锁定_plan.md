# 对话人物移动锁定 — 设计方案

## 需求理解

当前玩家正式对话已经能暂停后端时钟，并让 `LerpMovementProvider` 在 `GameManager.IsDialogueActive` 时冻结移动。但 NPC 与 NPC 的世界气泡对话期间，前端仍可能收到行为移动或社交动作移动，导致人物在对白显示时继续乱动。

本轮目标：

1. 玩家与 NPC 正式对话期间，被对话 NPC 也要有明确移动锁。
2. NPC 与 NPC 头顶气泡对话期间，双方都不能继续移动。
3. 对话期间收到的新移动不要丢失，保留最后一次待执行移动，等解锁后再执行。
4. 不扩后端协议，不改变当前 WebSocket 消息结构。

## 方案

### 1. `NpcEntity` 增加移动锁

新增：

1. `IsMotionLocked`
2. `PushMotionLock(reason)`
3. `PopMotionLock(reason)`
4. 锁定期间 `MoveToLocation()` 不立即移动，只记录最后一次 pending location。
5. 解锁后自动执行最后一次 pending move。

这样可以覆盖：

1. 后端行为消息触发的 `NpcSpawner.HandleNpcBehavior()`
2. NPC 社交动作触发的 `NpcBubbleManager.HandleNpcSocialAction()`
3. 其他未来调用 `NpcEntity.MoveToLocation()` 的入口

### 2. `LerpMovementProvider` 识别 NPC 移动锁

当前移动协程只看 `GameManager.IsDialogueActive`。

本轮增加：

1. 若 Transform 上挂有 `NpcEntity`
2. 且 `npc.IsMotionLocked`
3. 则协程暂停，不累计 elapsed

这样可以冻结“已经走到一半”的 NPC。

### 3. `NpcBubbleManager` 管理会话锁

对同一组 NPC 对话 key：

1. 第一条气泡开始播放时，对 speaker + target 加锁。
2. 队列播完后释放双方锁。
3. 玩家正式对话清理某 NPC 气泡队列时，也释放相关会话锁，避免锁泄漏。
4. `NPC_SOCIAL_ACTION` 不再为了对话强行移动发起者，避免气泡前后人物乱跑。

## 风险点

1. 如果气泡队列异常中断，锁需要可靠释放。
2. pending move 只保留最后一次位置，可能跳过中间移动路径；但这符合“对话期间不要乱动”的体验目标。
3. 当前不做面向目标 NPC 的朝向/站位系统，后续如果要“先靠近再开聊”，应单独做社交站位阶段。
