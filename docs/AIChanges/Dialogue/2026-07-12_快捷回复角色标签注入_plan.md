# 快捷回复角色标签注入 — 设计方案

## 需求理解

玩家快捷回复建议目前只拿到 NPC 名字、当前语气、最近对话和玩家记忆，缺少角色基础信息与视觉标签。

这会导致 LLM 根据名字或温柔/文艺等语气错误猜测性别，例如把男性角色“和叶”生成成“和叶姐”“她”“一点都没变”一类错位回复。

## 方案思路

1. 在 5 个 NPC profile 中补充：
   - `visual_tags`：面向提示词的稳定视觉/身份标签
   - `address_hint`：玩家快捷回复可用称呼与禁用称呼边界
2. 在 `PromptBuilder.build_player_reply_suggestions(...)` 中注入当前 NPC 基础信息：
   - 名字、年龄、性别、身份、视觉标签、称呼边界
3. 强化玩家快捷回复建议系统规则：
   - 必须以基础信息为准
   - 不能凭名字或说话风格推断性别
   - 不要生成“姐姐/姐/她”等与男性角色冲突的称呼

## 涉及文件

- `backend/config/npc_profiles/sakura.json`
- `backend/config/npc_profiles/chihaya.json`
- `backend/config/npc_profiles/kazuha.json`
- `backend/config/npc_profiles/tatsunosuke.json`
- `backend/config/npc_profiles/kujo.json`
- `backend/src/dialogue/prompt_builder.py`

## 风险点

1. Prompt 过长：只注入一行紧凑基础档案，避免影响速度。
2. 称呼过死：`address_hint` 提供边界，不强制每条都使用固定称呼。
3. 配置字段兼容：新字段只被快捷回复读取，旧逻辑不依赖，不会破坏现有冷启动。
