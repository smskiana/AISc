# ADR-0005: 项目文档采用功能目录优先索引

## 状态

Accepted

## 背景

项目已有 Workstream 和 ChangeIndex，但大量执行记录仍平铺在 `docs/AIChanges/` 根目录，并通过日期文件名和最新 handoff 导航。该结构使 AI 容易把时间接近误判为业务相关，也无法从目录本身理解内容边界。

## 决定

1. 项目索引和文档目录以功能域作为第一分类维度。
2. 每个受维护目录使用 `README.md` 说明文件夹功能、内容和入口。
3. `docs/AIChanges/` 的 plan / execution 按主要功能域存放。
4. 日期可以保留在执行证据文件名中用于审计，但不得作为目录层级或索引分组。
5. handoff 统一归档，不再作为默认上下文入口。
6. AI 查找内容时遵循“功能索引 -> 当前口径 -> 代码目录 -> 必要历史证据”。

## 与 ADR-0004 的关系

本 ADR 补充 ADR-0004 的结构化治理方式，不推翻“暂不建设文档向量库”的决定。`rg`、功能 README、Workstream 和 ADR 仍是主要定位机制。

## 影响

1. 新功能目录必须同步提供 README。
2. 新执行记录不得放回 `docs/AIChanges/` 根目录。
3. 旧路径迁移后必须同步修复 Markdown 引用并执行链接检查。
4. 当前口径仍以 Workstream、ADR 和代码为准，AIChanges 只提供实现证据。
