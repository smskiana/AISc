# 架构决策记录索引

## 作用

ADR 用来记录已经收敛的架构决定，避免后续会话反复推翻近期刚定下来的方向。

ADR 不是执行记录，也不是详细设计文档。它只回答：

1. 当时为什么需要决定。
2. 最终决定是什么。
3. 放弃了哪些替代方案。
4. 这个决定带来的约束是什么。
5. 什么时候可以重新讨论。

## 维护规则

1. 不直接删除旧 ADR。
2. 如果要推翻旧决定，新增 ADR 并写明 supersedes。
3. 如果只是补执行证据，更新工作流或 ChangeIndex，不改 ADR。
